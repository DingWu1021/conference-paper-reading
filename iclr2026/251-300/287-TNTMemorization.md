# TNT: Improving Chunkwise Training for Test-Time Memorization

> **Venue**: ICLR 2026 | **ArXiv**: 2511.07343 | **TNTMemorization**

近年来，基于循环神经网络（RNN）的序列模型在研究社区重新引发了广泛关注。以 Titans 和 TTT（Test-Time Training）为代表的深度测试时记忆模块（deep test-time memorization modules）展示了一种与 Transformer 截然不同的线性扩展范式：这类模型在处理长序列时，计算复杂度不随序列长度平方增长，因此在理论上具备优越的可扩展性。然而，这类颇具潜力的架构在实际落地中面临一个根本性瓶颈——训练速度极慢，硬件利用率极低（现有方法下，峰值硬件性能的利用率不足 5%～10%），导致其优势无法充分发挥。现有的并行化方案均依赖于一个关键超参数——分块大小（chunksize），该参数造成了训练效率与模型精度之间难以调和的冲突：较大的分块尺寸可以加速训练并提升硬件利用率，但会损害模型捕获细粒度序列依赖关系的能力；较小的分块尺寸则可以保留更精细的学习信号，但会引入大量串行依赖，严重拖累训练吞吐量。研究者们长期困于这一固定且次优的折中选择，本文正是针对这一根本性矛盾提出了解决方案。

本文提出的方法名为 TNT（Two-stage training with TNT），其核心思路是将训练效率与推理精度彻底解耦，通过两阶段的训练范式同时获得高训练速度和高模型质量。TNT 的技术创新体现在两个相互协作的组件上：层次化记忆架构（hierarchical memory architecture）和 Q-K 投影机制（Q-K Projection）。

在层次化记忆架构中，TNT 引入了全局记忆模块（global memory module）和多个并行本地记忆模块（local memory modules）的组合设计。全局记忆模块负责处理大块（如 $C_G = 2048$ 个 token）的序列片段，通过标准的分块梯度下降更新快权重矩阵 $V$，以较低的计算开销捕获长程上下文依赖。而多个并行本地记忆模块则专注于细粒度的局部信息处理：每个本地模块在固定长度的序列碎片（shard）边界处将记忆状态 $W_t$ 重置为可学习的初始状态 $W_{\text{init}}$。这一周期性重置操作是架构的关键创新——它从根本上打破了非线性 RNN 中固有的序列依赖链，使得各本地模块可以在不同设备上并行计算，从而实现大规模的上下文并行化（context parallelization）。在数学形式上，本地记忆的更新方程为在每个 shard 起始处令 $W_t \leftarrow W_{\text{init}}$，由此消除了碎片间的因果依赖。多个本地模块可同时以不同分辨率运行，形成多尺度时序建模能力，整体架构的运行时复杂度相对序列长度保持线性增长。

Q-K 投影机制针对的是深度记忆模块中一个较少被注意到但同样重要的问题——压缩域与检索域之间的不匹配（compression-retrieval domain mismatch）。这类记忆模块在训练时将键值对 $(k, v)$ 压缩存入快权重中，但推理检索时输入的是查询向量 $q$，两者来自不同分布，直接导致记忆检索质量下降。TNT 通过将查询向量投影到观测键向量所张成的子空间来缓解这一不匹配，其投影公式为

$$\hat{q}_t = \sum_{\tau} \frac{k_\tau k_\tau^\top}{\|k_\tau\|^2} q_t$$

该投影矩阵通过滑动累积方式维护，保持常数大小，计算开销极低。该机制仅应用于对域偏移更敏感的本地记忆模块，在消融实验中验证了其必要性：去除 Q-K 投影后，语言模型困惑度（perplexity）从 21.04 上升至 22.01。

第一阶段为效率导向的预训练（efficiency-focused pre-training），采用上述层次化记忆结构，以大块全局更新和并行化本地更新协同运行，训练阶段实现大规模并行；第二阶段为精度导向的微调（performance-focused fine-tuning），仅对本地记忆模块进行短暂微调，将本地分块大小 $C_L$ 降至更小的值（最低可至 $C_L = 1$），以极小的额外计算代价（约仅占预训练总计算量的 5%）换取模型在细粒度序列建模上的质量提升。

实验设计全面覆盖了训练效率和模型质量两个维度。模型规模为 1.5 亿参数，使用 T5 分词器（词表大小 32k），以 AdamW 优化器和余弦学习率调度（峰值学习率 $1 \times 10^{-3}$）在 TPUv5 集群上训练，数据集包括 C4、FineWeb 和 PG19。在训练速度方面，TNT 相比最精确的 Titans 基线配置（分块大小 $C = 8$）实现了高达 17.37 倍的加速，且在序列长度从 2k 扩展至 32k 的过程中，运行时间保持线性增长，验证了架构的可扩展性。在模型质量方面，TNT 第一阶段（全局分块 $\{4,8,16,32\}$）在 C4/FineWeb/PG19 的平均困惑度为 23.13，较基线 Titans 的 27.13 有显著改善，甚至接近使用 FlashAttention 的门控 Transformer（22.39）；经过第二阶段微调后（本地分块 $\{2,4,8,16\}$），平均困惑度进一步降至 23.09，常识推理准确率（在 PIQA、HellaSwag、ARC-e、CommonsenseQA 四个任务上）从基线的 38.8% 提升至 40.9%，超越 Transformer 基线（39.7%）。消融实验进一步证实了各组件的贡献：逐步增加本地模块数量（从 0 到 4 个）使困惑度从 23.53 稳定下降至 20.15；移除全局记忆模块导致困惑度大幅退化至 25.60；第二阶段微调额外贡献了约 0.76 个困惑度单位的改善。

综合来看，TNT 的研究意义体现在多个层面。从工程角度，它将深度测试时记忆模块的训练速度提升了一个数量级，首次使这类架构的大规模训练在计算上变得可行，为后续探索更大规模、更长上下文的 RNN 类模型扫清了障碍。从方法论角度，两阶段"粗粒度预训练 + 细粒度微调"的解耦范式具有普适性，理论上可推广到任何依赖分块策略进行并行化的序列模型。从模型理解角度，TNT 揭示了分块大小对测试时记忆模型的性能影响并非单调关系——并非推理时分块越小越好，而是需要结合训练时的分辨率设置统一优化——这一发现修正了领域内此前的直观认知。此外，Q-K 投影机制所揭示的记忆压缩-检索域不匹配问题，为未来改进记忆型模型的检索质量提供了新的研究视角。总体而言，TNT 在效率和精度上同时取得突破，为推动 RNN 类线性复杂度模型走向实用提供了重要的技术基础。
