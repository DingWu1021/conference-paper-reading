# Locality-aware Parallel Decoding for Efficient Autoregressive Image Generation

> **Venue**: ICLR 2026 Oral | **ArXiv**: 2507.01957 | **LocalityDecoding**

自回归图像生成在近年来取得了长足的进步，以LlamaGen、VAR等为代表的方法通过逐块（patch-by-patch）或逐token的下一步预测范式，实现了与扩散模型媲美的生成质量。然而，这一范式存在一个固有的效率瓶颈：生成过程本质上是串行的，每一步只能生成一个或少数几个token，这导致整个生成流程是"内存受限"（memory-bound）的操作——处理器的大量时间耗费在从内存加载模型权重上，而非真正的计算，使得生成延迟极高。对于分辨率为256×256的图像，标准自回归模型需要256步推理；而512×512则高达1024步，这在实际部署中几乎是不可接受的。如何在不牺牲生成质量的前提下大幅减少推理步数，是高效自回归图像生成领域的核心挑战。

已有若干工作尝试通过并行解码来加速自回归生成，例如RAR、PAR、RandAR、ARPG、NAR等方法。这些方法的基本思路是将生成过程分组，每组内同时生成多个token。然而，它们普遍面临两个关键问题：其一，并行生成的token之间缺乏相互可见性（mutual visibility），导致同组内的token无法感知彼此，在高并行度时生成质量大幅下降；其二，生成顺序往往采用随机或简单的几何序列（如Halton序列），没有针对图像token之间的空间依赖关系进行优化，使得每步解码时可利用的上下文信息不充分。针对这两个问题，本文提出了局部感知并行解码（Locality-aware Parallel Decoding, LPD）方法，从架构设计和生成调度两个维度系统性地解决了并行自回归图像生成的质量与效率问题。

本文的核心技术贡献体现在两个相互配合的创新点上。

第一个贡献是**灵活并行化自回归建模**（Flexible Parallelized Autoregressive Modeling）架构。传统自回归建模将联合概率分解为 $p(x_1,\ldots,x_N;c) = \prod_n p(x_n \mid x_{<n};c)$，要求严格的串行生成。LPD将其推广为基于分组的并行生成：

$$p(x_1,\ldots,x_N;c) = \prod_g p(X_g \mid X_{<g};c)$$

其中 $X_g$ 表示第 $g$ 组内所有并行生成的token集合，$X_{<g}$ 表示之前所有组已生成的token。实现这一并行化的关键机制是**可学习位置查询token**（learnable position query tokens）。在推理时，已生成的token通过KV缓存提供上下文表示，而位置查询token则携带目标生成位置的位置编码信息，引导模型在目标位置并行生成新token。同组内的查询token之间可以相互可见（mutual visibility），从而确保并行生成的token能够形成一致的输出，避免了之前方法中同组token"各自为政"导致的不一致性问题。在训练时，模型使用特殊设计的注意力掩码将真实token与位置查询token交织在一起：上下文注意力（context attention）使后续token能因果地关注上下文，而查询注意力（query attention）则使同组的查询token相互可见但阻止后续token反向关注查询token，从而在训练中模拟并行解码的行为。在推理时，已采样token被编码后与位置查询token融合操作，KV缓存中只保存实际token状态而非查询token状态，兼顾了效率与一致性。

第二个贡献是**局部感知生成顺序**（Locality-aware Generation Ordering）调度算法。作者首先通过分析LlamaGen的注意力模式，发现图像token之间存在强烈的空间局部性——每个token的注意力权重高度集中于其空间近邻，并以"每token注意力"（Per-Token Attention, PTA）指标定量刻画：$\text{PTA}_s$ 衡量token对欧氏距离为 $s$ 处其他token的平均注意力，结果显示注意力随距离增大而显著衰减。基于这一洞察，作者设计了一个分组调度算法，遵循两个相互平衡的原则：（1）**高近邻上下文支持**——新token应选择在已生成token的空间近邻处，以获得充分的局部上下文；（2）**低组内依赖**——同组内并行生成的token之间应尽量保持空间距离，减少相互依赖。具体地，算法通过计算基于欧氏距离的近邻分数，先选出与已有上下文高度近邻的候选token（近邻阈值 $\tau$），再通过排斥阈值 $\rho$ 过滤掉组内过近的token对，最后用最远点采样（farthest-point sampling）补齐每组的slot，从而在充分利用空间上下文的同时保证组内并行token的独立性。

在实验设计上，作者在标准ImageNet类别条件生成基准（256×256和512×512）上系统评估了LPD，并与扩散模型（DiT-XL/2、SiT-XL/2）、掩码预测模型（MaskGIT、MAR、MaskBit）、VAR以及各类并行自回归方法（RAR、PAR、RandAR、ARPG、NAR）进行全面比较。LPD系列模型包括LPD-L（337M参数）、LPD-XL（752M参数）和LPD-XXL（1.4B参数），均基于LlamaGen分词器（码本大小16384，下采样率16×）构建，采用标准的仅解码器Transformer架构。

定量结果十分突出。在256×256分辨率下，LPD将生成步数从256步压缩至20步（压缩比12.8×），LPD-XL实现FID 2.10，延迟仅0.41秒，吞吐量75.20 images/s；LPD-XXL达到FID 2.00，延迟0.55秒。相比之下，先前最优并行自回归模型ARPG-XXL在64步下FID为1.94但延迟高达2.24秒，标准串行基线（256步）延迟7.10秒。LPD相比先前并行方法实现了至少3.4倍的延迟降低。在512×512分辨率下，LPD将生成步数从1024步减少至48步（压缩比21.3×），LPD-XL以1.01秒实现FID 2.10，而串行基线需要20.93秒。此外，作者还在内部1000万图像数据集上训练了文本生成图像（text-to-image）模型，在GenEval等指标上验证了LPD在1024×1024高分辨率生成任务中的可扩展性。

消融实验进一步揭示了各组件的独立贡献。相互可见性机制的关键性体现在：在高并行度（32步）下，LPD的FID几乎与256步基线持平，而RandAR和ARPG则显著退化，说明组内token的相互感知对维持质量至关重要。生成顺序的有效性通过对比实验得到验证：局部感知顺序（FID 1.92，32步）明显优于随机顺序（2.11）和Halton几何序列（2.06）。两个调度原则的分解实验表明：仅使用原则一（最大化上下文近邻支持）得FID 2.00，仅使用原则二（最小化组内依赖）得FID 2.06，两者结合达到最优FID 1.92，体现了协同效应。超参数敏感性分析表明，近邻阈值 $\tau$ 和排斥阈值 $\rho$ 在默认值（$\tau=2$, $\rho=1$）附近均不敏感，算法鲁棒性良好。效率分析显示，系统在小批量（batch≤16）时处于内存受限状态，位置查询token虽引入额外开销，但生成步数的减少几乎线性地转化为延迟降低；即便在最大批量下进入计算受限状态，LPD仍保持约3倍的吞吐量优势。

综合来看，LPD的工作意义在于从根本上重新审视了并行自回归图像生成的两个核心问题——"如何并行"与"按什么顺序生成"——并给出了原理清晰、效果显著的解决方案。相互可见性机制与局部感知调度的结合，使得并行化程度的大幅提升不再以牺牲生成质量为代价，而是通过更合理的信息流设计来保障质量。这一思路对于其他序列生成任务（如视频生成、语言模型推理加速）也具有重要的启发价值。该工作作为ICLR 2026的口头报告（Oral）入选，代码已在MIT Han Lab的GitHub仓库（mit-han-lab/lpd）开源，展示了研究社区对这一方向高度的认可与关注。未来，如何将LPD的思想扩展到连续值自回归生成（如MAR框架）、以及如何与投机解码等其他加速技术相结合，是值得深入探索的方向。
