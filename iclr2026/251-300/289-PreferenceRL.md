# Policy Likelihood-based Query Sampling and Critic-Exploited Reset for Efficient Preference-based Reinforcement Learning

> **Venue**: ICLR 2026 | **ArXiv**: N/A | **PreferenceRL**

基于人类偏好的强化学习（Preference-based Reinforcement Learning，PbRL）是近年来机器人控制和连续控制领域备受关注的研究方向。与传统强化学习依赖人工精心设计奖励函数不同，PbRL 通过采集人类对智能体行为轨迹的偏好反馈来训练奖励模型，进而引导策略优化。这一范式的核心优势在于无需专业知识即可描述复杂的行为目标，但其实际效果高度依赖于查询采样策略的质量——即如何从智能体已经历过的轨迹中选取具有代表性的轨迹对，呈现给人类标注者以获取偏好反馈。在这一背景下，本文提出了 PoLiCER（Policy Likelihood-based Query Sampling and Critic-Exploited Reset），系统性地解决了现有方法在查询有效性和奖励估计偏差两方面的核心瓶颈。

现有 PbRL 方法在查询采样环节普遍存在一个被忽视的问题：绝大多数方法从智能体的历史回放缓冲区（replay buffer）中随机或按照某种启发式规则抽取轨迹对，而随着训练的推进，早期储存在缓冲区中的轨迹往往对应智能体早期低水平的行为模式，与当前策略的行为分布之间存在显著偏移。这些"过时"轨迹在当前策略下具有极低的似然度（likelihood），它们所携带的人类偏好信号对于指导当前策略的改进几乎没有信息价值。换言之，人类标注者在比较两段行为轨迹时，其反馈本应帮助智能体区分"在当前能力边界上哪种行为更优"，但如果参与比较的轨迹均来自历史早期、与当前策略脱节，那么所获得的偏好标签对当前训练阶段的参考意义便大打折扣。本文将这一现象概括为查询与策略的不对齐问题（query-policy misalignment），并将其视为制约 PbRL 效率的根本原因之一。

为解决上述问题，PoLiCER 提出了基于策略似然的查询采样机制（Policy Likelihood-based Query Sampling，PLS）。其核心思想是，在从缓冲区选取轨迹时，优先选择那些在当前策略下具有较高轨迹似然度的片段。具体而言，对于缓冲区中候选的轨迹片段 $\tau = (s_1, a_1, \ldots, s_L, a_L)$，其在当前策略 $\pi_\theta$ 下的对数似然可表示为

$$\log p_{\pi_\theta}(\tau) = \sum_{t=1}^{L} \log \pi_\theta(a_t \mid s_t)$$

PLS 依据此似然分数对缓冲区中的轨迹进行排序或加权采样，确保进入查询集的轨迹与当前策略的行为分布高度一致。由于只需对候选轨迹执行前向传播即可计算似然分数，算法的额外计算开销极小：对于包含 $L$ 步、每次采样 $K$ 条轨迹的配置，整个 PLS 过程仅需 $2 \times L \times K$ 次前向传播，而基于集成模型的分歧度采样（disagreement-based sampling）所需的开销则是其 $N$ 倍（$N$ 为集成模型数量）。这使得 PLS 在保证采样质量的同时，具备优良的计算效率。

然而，PLS 机制在带来查询对齐收益的同时，也引入了一个新的风险：当采样高度聚焦于当前策略覆盖的状态-动作空间区域时，奖励模型所接收到的训练信号会越来越集中于某一局部区域，导致奖励估计的覆盖范围不断收窄。更严重的是，由于奖励估计器对早期反馈数据的过拟合（primacy bias），Q 函数在这些局部区域容易产生过高的累积奖励估计，即所谓的过估计偏差（overestimation bias）。这种偏差一旦形成，便会通过 Bellman 更新链式传播，导致策略被错误地引导至被高估区域，严重影响后期的探索能力和泛化性能。

为应对这一隐患，PoLiCER 引入了批评者引导的重置机制（Critic-Exploited Reset，CER）。CER 的核心洞察在于：Q 函数（即批评者）的输出值本身蕴含了关于奖励过估计程度的信息。当 Q 值异常偏高时，往往意味着奖励估计器已经陷入了局部过拟合状态，此时对奖励估计器及其关联的 Q 函数实施参数重置可以有效打破这一恶性循环。形式上，CER 监控当前 Q 值均值 $\bar{Q}_t$，当其超过某一动态阈值时，触发对奖励估计器和 Q 网络的选择性重置。重置后，网络从随机初始化或部分保留的参数状态重新学习，从而拓宽奖励信号的覆盖范围，消除累积的估计偏差。这一设计的精妙之处在于，重置并非盲目地定期执行，而是由 Q 函数自身的异常信号驱动，既避免了不必要的重置带来的学习效率损失，又能在偏差真正积累到危险水平时及时干预。

从整体架构来看，PLS 与 CER 两个组件形成了互补的协同关系：PLS 确保人类反馈信号始终对准当前策略，提升反馈信息量；CER 则在 PLS 的精确聚焦可能导致过拟合时发挥"重置阀门"的作用，维持奖励估计的全局可靠性。两者共同作用，使得 PoLiCER 在有限的人类查询预算下能够比现有方法更高效地利用每一次偏好反馈。

在实验评估方面，PoLiCER 在两个主流连续控制基准平台上进行了系统验证：DeepMind Control Suite（DMC）的多个运动控制任务（包括 walker-walk、cheetah-run、quadruped-walk 等高维运动任务）以及 Metaworld 的多个机器人操作任务（包括 button-press、door-open、sweep-into 等精细操作场景）。实验对比了多种代表性 PbRL 基线方法，涵盖 PEBBLE（基于集成不确定性的查询采样）、B-Pref（多种查询策略的基准评测框架）、SURF（使用数据增强提升样本效率）、RUNE（基于奖励不确定性的探索奖励）、MRN（基于相互信息的奖励网络）以及 QPA（查询先验对齐方法）。实验结果表明，PoLiCER 在几乎所有测试任务上均取得了最优或接近最优的性能，尤其在查询预算极为有限的设置下（即只能获取少量人类标注的情况），PoLiCER 的优势更为突出，这正体现了策略对齐查询与过估计偏差缓解两方面改进的综合收益。此外，作者还在合成标注者（synthetic annotator，模拟人类噪声偏好）和人类在环（human-in-the-loop）两种设置下均验证了方法的有效性，表明 PoLiCER 对标注噪声具有良好的鲁棒性。

消融实验进一步印证了两个组件各自的必要性。仅使用 PLS 而不加 CER 时，模型在长时间训练后性能出现明显下滑，印证了单独使用策略对齐采样会引发过拟合问题的假设；仅使用 CER 而不结合 PLS 时，性能改善有限，说明若查询本身缺乏信息量，仅靠重置机制无法充分提升学习效率；两者结合的完整 PoLiCER 则在性能和稳定性上均表现最优。

综合而言，PoLiCER 在偏好强化学习领域做出了两项具有实践价值的原则性贡献。第一，它明确指出并量化了查询-策略不对齐现象对反馈信息量的负面影响，并提出了一种计算高效的解决方案；第二，它揭示了策略对齐采样与奖励过估计之间的内在联系，并通过批评者驱动的自适应重置机制优雅地化解了这一矛盾。从更宏观的视角看，PoLiCER 的思路对 RLHF（基于人类反馈的强化学习）等大语言模型对齐技术同样具有启示意义：在 LLM 的偏好学习场景中，如何保证用于训练奖励模型的对比数据与当前模型输出分布保持一致，是提升对齐效率的关键问题，而 PoLiCER 提供的策略似然评分框架或许可以为此提供借鉴。该工作的代码已开源（https://github.com/JongKook-Heo/PoLiCER），为后续研究提供了可复现的基础。
