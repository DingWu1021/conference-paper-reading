# Speculative Actions: A Lossless Framework for Faster AI Agents

> **Venue**: ICLR 2026 Oral | **ArXiv**: 2510.04371 | **SpeculativeActions**

当前基于大语言模型（LLM）的智能体系统在真实环境中执行任务时，面临严重的延迟瓶颈。智能体的每一步行动通常需要向外部环境或后端服务发起 API 调用，而这些调用的响应时间往往达到数秒甚至更长。由于传统智能体框架采用顺序执行范式——必须等待当前步骤的 API 响应后才能决策下一步行动——整体任务完成时间随轨迹长度线性增长。论文以两个高水平国际象棋 AI 对弈为例：即便双方都使用顶级推理模型，完成一局比赛也可能耗费数小时。这一现象在智能体训练、评估和在线部署中普遍存在，严重制约了 AI 智能体在实际应用中的效用。面对这一挑战，来自哥伦比亚大学的研究团队（Naimeng Ye、Arnav Ahuja、Georgios Liargkovas、Yunan Lu、Kostis Kaffes、Tianyi Peng）借鉴微处理器领域的推测执行（Speculative Execution）技术以及大语言模型推理领域的推测解码（Speculative Decoding）思想，提出了一套适用于通用智能体系统的无损加速框架——推测行动（Speculative Actions）。

推测行动框架的核心思想是将智能体的行动执行过程从串行改造为并行。框架引入两类角色：**执行者（Actor）**和**推测者（Speculator）**。执行者是权威但较慢的主体，可以是功能强大的 LLM、外部 API 或人类用户；推测者则是一个快速、廉价的模型，其任务是在执行者尚未返回响应期间，预测执行者最可能给出的回应，并提前发起后续步骤的 API 调用。当执行者的真实响应到达后，系统检查已预发起的推测分支中是否存在与真实响应匹配的分支：若匹配，则直接提交该分支的结果，从而节省了原本需要等待后续 API 调用完成的时间；若不匹配，则丢弃错误的推测分支，回退到正确状态并重新执行。由于每次提交均以执行者的真实响应为准，整个框架在功能上保持无损性——最终输出与纯顺序执行完全等价。

在形式化建模层面，论文将智能体系统抽象为马尔可夫决策过程（MDP），其中每个行动代表一次 API 调用。在时间步 $t$，策略 $\pi$ 将状态 $s_t$ 映射为历史信息 $h_t$ 和查询 $q_t$，行动异步执行并返回一个 future，状态转移由函数 $f$ 完成：$s_{t+1} \leftarrow f(s_t, a_t)$。算法的核心流程为：（1）向执行者发起真实调用；（2）同时运行推测者生成 $k$ 个候选预测；（3）为每个候选预测预先发起下一步的 API 调用；（4）当执行者响应到达时，提交与之匹配的推测分支。论文给出了理论分析（命题1），证明在推测准确率为 $p$、推测者延迟为 $\alpha$、被推测的 API 调用延迟为 $\beta$ 的条件下，期望运行时间之比收敛为 $1 - \frac{p}{1+p} \cdot \frac{\alpha}{\alpha + \beta}$。当 $p = 1$ 且 $\alpha \gg \beta$ 时，理论上最多可实现 50% 的延迟降低。为确保安全性，框架设计了三类保护机制：语义守卫（Semantic Guards，验证状态转移等价性后方可提交）、安全信封（Safety Envelopes，仅允许幂等、可逆或沙箱化的推测操作）以及修复路径（Repair Paths，预测失败时执行回滚或补偿动作）。框架还支持多步推测树（预测未来多步形成树状结构）和自适应推测（基于置信度估计进行束搜索式优先级排序）等扩展形式。

实验在四个多样化领域展开，全面验证了框架的通用性与有效性。在国际象棋对弈环境中，论文让两个 GPT-5 智能体对弈，轮到对方行动时本方智能体使用降低推理力度的 GPT-5 作为推测者，预测对方最可能的走法并提前计算回应。实验结果显示，在 3 个推测分支的设置下，预测准确率达到 54.7%，平均节省时间 19.5%，同时还实现了平行度与 token 代价之间的量化权衡分析。在电商场景下，系统在用户尚未完成输入时预测其完整查询意图及所需 API 调用（如 `get_user_details`、`get_order_details`），基于 $\tau$-bench 零售基准评测，不同模型组合下 API 预测准确率为 22%–38%，多模型推测者组合优于单一模型，低成本推测者仅需 2–3 秒即可完成，而典型用户输入时间约 30 秒，使得约三分之一的交互可实现近实时响应。在多跳问答（HotpotQA）场景下，智能体通过顺序维基百科检索回答复杂问题，推测者提前预测下一次检索所需的文章内容，top-3 预测准确率达到 46%；值得注意的是，更强的执行者模型反而会降低预测准确率，原因在于强模型的输出更加多样，增加了精确匹配的难度。在操作系统超参数调优场景（有损扩展）中，系统联合运行推测者和执行者以调整 Linux CFS 调度器的 `min_granularity` 参数，最小化 CPU 负载下的 p95 延迟：单独使用推测者约 20 秒收敛但停留在次优配置（36.24ms），单独使用执行者约需 200 秒且容易陷入局部最优（54.00ms），而两者协同仅需 10–15 秒即达到最优设置（0.2ms），平均延迟降至 37.93ms，展现出远超各自单独运行的优势。

这项工作在理论和工程两个维度均具有重要意义。从理论层面看，推测行动将推测执行的范式从芯片微架构和 LLM 令牌级推理扩展到了智能体行动级，给出了严格的无损性证明和量化的延迟-代价分析框架，为后续研究奠定了形式化基础。从工程层面看，论文提出的框架与具体智能体架构和任务领域解耦，只需在现有系统中引入轻量推测者即可部署，且当使用自托管 LLM 时，并行 API 调用的额外计算成本可通过批处理机制大幅摊薄。对于搜索增强推理和多智能体协作等研究方向而言，推测行动框架尤其具有启发价值：在需要多轮检索或跨 Agent 通信的场景中，顺序等待往往是主要延迟来源，而基于预测的预执行策略可以从根本上打破这一瓶颈。未来研究方向包括不确定性感知的推测策略、分层的执行者-推测者架构设计，以及与强化学习训练循环的深度集成，有望进一步提升智能体系统在实时交互环境中的整体效能。
