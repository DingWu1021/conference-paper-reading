# MemGen: Weaving Generative Latent Memory for Self-Evolving Agents

> **Venue**: ICLR 2026 | **ArXiv**: 2509.24704 | **MemGen**

在以大型语言模型（LLM）为核心的智能体系统飞速发展的背景下，如何赋予智能体有效的记忆能力始终是一个核心挑战。现有的记忆范式大体可分为两类：参数化记忆与检索式记忆。参数化记忆通过微调或持续学习直接调整模型参数，将经验编码进权重之中；检索式记忆则将历史经验外化为结构化数据库，在推理时通过相似性检索召回相关信息并拼接进上下文。然而，这两种范式均存在明显局限。参数化记忆的更新成本高昂，且容易导致灾难性遗忘，难以灵活应对动态变化的任务环境；检索式记忆依赖离散的文本片段，信息颗粒度粗，检索噪声大，且无法真正融入模型的内部推理流程之中。更根本的缺陷在于，这两种范式均未能模拟人类认知中记忆与推理相互编织、动态交融的本质特征——人类在思考过程中会随时触发记忆的提取与更新，记忆本身也随思维的深入而不断演化，二者绝非割裂的两个阶段。正是基于这一观察，来自新加坡国立大学的张贵彬、傅沐鑫和颜水成提出了MemGen框架，旨在为LLM智能体赋予一种更加接近人类认知模式的、动态生成式的潜在记忆机制。

MemGen的核心创新在于将记忆的生成与召回过程直接嵌入模型的推理流之中，以潜在令牌序列（latent token sequence）的形式表达机器原生的记忆内容。与依赖自然语言文本的外部记忆不同，MemGen的记忆以连续的隐层表示存在，能够承载更丰富、更精确的语义信息，同时与语言模型的内部表示空间天然兼容。框架由两个核心模块构成：记忆触发器（Memory Trigger）和记忆编织器（Memory Weaver），二者均以LoRA适配器的形式附加于冻结的基础语言模型之上，实现轻量级的参数高效学习。

记忆触发器的职责是在每个生成步骤动态判断是否需要显式调用记忆。在生成第 $j$ 个令牌时，触发器以当前步骤之前的隐状态序列 $H_{t,<j}$ 为输入，计算一个二元调用概率：

$$p_j = \sigma\left(T_{\text{trigger}}\left(H_{t,<j}\right)\right)$$

其中 $\sigma$ 为sigmoid函数，$T_{\text{trigger}}$ 为触发器的LoRA映射。系统根据 $p_j$ 采样一个二元决策——调用（INVOKE）或跳过（SKIP），仅当决策为INVOKE时才激活记忆编织器。这一自适应触发机制避免了在每个推理步都强制进行记忆召回所带来的计算冗余，使记忆的介入与推理状态保持动态对齐。

记忆编织器在被触发后接收当前推理状态，生成一段固定长度的潜在令牌矩阵作为记忆内容：

$$M_t = W_{\text{weaver}}\left(H_{t,<j}\right)$$

生成的潜在记忆令牌 $M_t$ 随后被插入后续的解码上下文之中，直接影响模型对后续令牌的生成概率分布。此外，记忆编织器还支持引入外部嵌入 $E_t$（例如来自知识库或历史轨迹的向量表示）以进一步丰富记忆内容：

$$M_t = W_{\text{weaver}}\left(\left[H_{t,<j};\, E_t\right]\right)$$

这种设计使得MemGen既能够以纯内生方式从自身推理状态生成记忆，也能够有机融合外部信息源，具备良好的扩展性。整个生成流程形成了一个"推理→触发→编织→再推理"的紧密闭环，记忆与认知真正交织在一起，实现了动态的自进化循环。

在训练策略上，MemGen采用分阶段训练方案。第一阶段对记忆编织器进行监督微调（SFT），使其学习如何根据当前推理状态合成有意义的潜在记忆令牌；第二阶段再对记忆触发器进行训练，使其学习在何时调用记忆最为恰当。两个模块均同时支持监督微调和基于群体相对策略优化（GRPO，Group Relative Policy Optimization）的强化学习训练，后者能够在没有显式监督信号的情况下，通过任务完成奖励引导模型自主探索最优的记忆触发与生成策略。整个框架基于Qwen2.5-1.5B-Instruct和SmolLM3-3B等小型基础模型实现，采用ChatML格式的对话模板，并支持分布式数据并行（DDP）训练，体现了良好的工程实用性。

实验方面，MemGen在覆盖多个认知能力维度的八个基准测试上进行了系统评估，涵盖代码生成（KodCode）、数学推理（GSM8K）、知识密集型问答（TriviaQA、GPQA）以及多轮交互决策（ALFWorld）等多种任务类型。实验结果令人印象深刻：相较于以ExpeL和AWM为代表的外部检索式记忆系统，MemGen在准确率上最高提升38.22%；相较于以GRPO为代表的纯强化学习基线，MemGen最高提升13.44%，展示出显著的性能优势。在跨域泛化实验中，在代码合成任务上训练的MemGen模型在数学推理任务上同样取得了明显提升，说明框架所学到的记忆生成能力捕捉到了领域无关的推理特征，具备良好的迁移潜力。

MemGen最引人注目的发现之一，是在完全没有显式监督的条件下，模型自发涌现出了三种类人记忆功能：规划记忆（Planning Memory）辅助多步推理与有序执行，帮助智能体在复杂任务中维持长程一致的行动计划；程序性记忆（Procedural Memory）则记忆工具调用规范与答案格式要求等反复出现的操作模式，减少重复性错误；工作记忆（Working Memory）负责在扩展序列中维护短期上下文的连贯性，确保推理过程不因信息遗忘而产生矛盾。这种从任务奖励信号中自然涌现的认知分工，与人类记忆系统中不同记忆类型的功能性分化高度吻合，揭示了MemGen作为智能体记忆研究范式的深层潜力。

综合来看，MemGen从根本上重新审视了LLM智能体记忆的形态与角色。通过将记忆从"外挂的信息仓库"重塑为"编织进推理流的潜在认知状态"，MemGen在设计哲学上更贴近认知科学对人类记忆的理解。这一工作不仅在多个基准上取得了强劲的性能提升，更通过涌现出的三类记忆功能为"什么样的记忆机制有利于LLM推理"这一基础问题提供了宝贵的实证依据。轻量级的LoRA实现方式和分阶段训练策略也保证了框架的可扩展性与实际部署可行性。未来，这一思路有望与更大规模的基础模型、更复杂的多智能体协作场景相结合，进一步推动具有持续自我进化能力的通用智能体系统的发展。
