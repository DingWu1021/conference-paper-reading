# SimpleTIR: End-to-End Reinforcement Learning for Multi-Turn Tool-Integrated Reasoning

> **Venue**: ICLR 2026 | **ArXiv**: 2509.02479 | **SimpleTIR**

工具集成推理（Tool-Integrated Reasoning，TIR）是近年来提升大语言模型（LLM）推理能力的重要范式：模型在生成回答的过程中可以调用外部工具（如 Python 解释器），借助工具执行的结果辅助完成复杂的数学或逻辑推断。然而，将 TIR 拓展到多轮交互场景并以强化学习（RL）端到端训练时，研究者面临严峻的训练不稳定与性能崩溃问题。现有工作大多绕开这一难题，选择先用监督微调（SFT）为模型提供一个"冷启动"的工具使用能力，再以 RL 进行精化，这种两阶段方案不仅流程繁琐，更会将 SFT 数据集的偏置引入最终策略，限制了模型自主探索多样推理行为的空间。SimpleTIR 的出发点正是要突破这一局限，实现从基础预训练模型出发、完全端到端的多轮 TIR 强化学习训练，同时彻底解决训练不稳定问题。

为理解训练崩溃的根本原因，作者从理论角度对梯度范数进行了系统分析。在基于群组相对策略优化（GRPO）的多轮 TIR 训练框架中，每个 rollout 轨迹由若干"轮次"（turn）组成：模型在每一轮生成一段文本，工具执行后将结果反馈给模型，如此往复。问题的核心在于"分布漂移"（distributional drift）：工具的外部反馈会将模型推向训练初期从未见过的输入分布，当模型对某些 token 分配极低的概率时，其对应的重要性比率（importance ratio）$\frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_\text{old}}(a_t|s_t)}$ 会趋于无穷大，进而引发梯度范数的灾难性爆炸。更严重的是，这种低概率生成会在多轮交互中不断累积放大——一旦某轮出现高随机性导致模型提前生成 EOS（序列结束）token 而中断推理流程，后续轮次的上下文将完全失控，形成"复合误差"（compounding errors）。作者将这类既不包含可执行代码块、也不包含最终答案的轮次称为"空洞轮"（void turn），并通过梯度分析证明，包含空洞轮的轨迹正是触发梯度爆炸的核心有害样本。

SimpleTIR 提出的解决方案简洁而高效：在策略梯度更新阶段，将所有包含空洞轮的完整轨迹从梯度计算中剔除，但在优势估计（advantage estimation）阶段仍然保留这些轨迹。这一设计的精妙之处在于两个方面：其一，通过阻断有害轨迹的梯度反传，彻底消除了梯度范数爆炸的来源，实现训练曲线的平稳收敛；其二，在优势估计时保留这些轨迹，保证了优势函数计算的无偏性——若完全丢弃这些轨迹，则其余轨迹的相对优势估计将产生系统性偏差，损害学习信号的质量。这种"更新时过滤、估计时保留"的非对称处理方式，使 SimpleTIR 成为一个即插即用（plug-and-play）的稳定化算法，可无缝嵌入现有的 GRPO 或 PPO 训练框架。

在训练框架的整体设计上，SimpleTIR 将多轮 TIR 建模为层次化马尔可夫决策过程（hierarchical MDP），轮次级策略与 token 级策略共享同一套神经网络参数，从而实现统一架构下的端到端训练。为确保模型只对自身生成的 token 负责而非工具返回的外部反馈，训练中引入了反馈遮蔽（feedback masking）机制，仅对模型输出部分计算损失。训练超参数方面，rollout 批量大小为 512，mini-batch 大小为 128，每个问题最多允许 5 到 10 轮工具交互，上下文窗口限制在 16K 至 24K token 之间，学习率设为 $1 \times 10^{-6}$，PPO 裁剪比率为 0.2 至 0.28。实验以 Qwen2.5-7B 和 Qwen2.5-32B 两个基础模型为起点，使用 Math3-5 和 Deepscaler 数据集进行训练，完全跳过任何形式的 SFT 冷启动阶段。

实验在多个具有挑战性的数学推理基准上进行了全面评估，包括 AIME24、AIME25、MATH500、OlympiadBench、AMC23 以及 HMMT25。与非 TIR 的零样本 RL 方法（SimpleRL-Zoo、DAPO）、冷启动 TIR 方法（ReTool、ToRL、Effective CIR）以及唯一另一个严格零样本 RL+TIR 方法 ZeroTIR 进行了对比。SimpleTIR-7B 在 AIME24 上取得 50.5 的得分，远超文本纯推理基线的 22.1 和 ZeroTIR 的 39.6；MATH500 得分 88.4，AMC23 得分 79.1，HMMT25 得分 29.7。SimpleTIR-32B 的表现更为突出：AIME24 达到 59.9，AIME25 达到 49.2，MATH500 达到 92.9，OlympiadBench 达到 63.7，在所有参与对比的方法中均名列前茅，且超越了多个基于更强指令微调模型训练的冷启动方法。消融实验进一步验证了空洞轮过滤策略的必要性：替换为低概率 token 过滤时 AIME24 仅得 23.3，替换为高重要性比率过滤时得 26.3，完全不过滤时仅得 20.8，均与 SimpleTIR 的 50.5 存在显著差距，且后三种方案均表现出明显的梯度范数尖刺，而 SimpleTIR 的训练曲线几乎没有任何波动。

SimpleTIR 的另一项重要发现是模型在无任何监督引导下涌现出多样且复杂的推理行为。研究者将观察到的推理模式归纳为三类：渐进式推理（Progressive Reasoning，即逐步分解问题并利用工具逐步求解）、交叉验证（Cross-Validation，即通过多种不同方法或代码实现对同一结果进行核验）以及错误修正（Error Correction，即模型识别出代码执行错误后主动调整策略重试）。SimpleTIR-32B 的渐进式推理频率高达 46.5%，远超冷启动方法的 18.9%；错误修正频率为 38.0%，高于对比方法的 25.8%。这表明，端到端的零样本 RL 训练相比 SFT 初始化反而能为模型保留更大的探索空间，让其自主发现更多元的解题策略。

综合来看，SimpleTIR 在方法论和实践两个层面均做出了重要贡献。方法层面，它以严格的理论分析揭示了多轮 TIR 训练不稳定的根本机制，并提出了原理清晰、实现简单的空洞轮过滤方案；实践层面，它证明了从基础模型出发、无需 SFT 冷启动即可实现高性能多轮工具集成推理的可行性，打破了"TIR 必须依赖 SFT 预热"的行业惯例。SimpleTIR 的即插即用特性使其可以方便地集成到任何现有的 RL 训练框架中，具备良好的推广价值。这项工作为后续研究多轮智能体推理、工具使用以及基于 RL 的复杂推理优化提供了坚实的基础，也为探索 LLM 自主涌现高级推理能力的机制开辟了新的方向。
