# UltraMemV2: Memory Networks Scaling to 120B Parameters with Superior Long-Context Learning

> **Venue**: ICLR 2026 | **ArXiv**: 2508.18756 | **UltraMemV2**

大语言模型在规模化过程中，稀疏计算已成为降低推理成本的关键技术路线。混合专家（Mixture of Experts，MoE）模型通过在每次前向传播时仅激活部分参数，实现了在维持总参数量不变的前提下提升模型容量的目标，但其核心缺陷在于推理阶段需要从内存中加载多个专家的权重，导致内存访问带宽（Memory Access Cost）成为系统瓶颈，尤其在边缘设备或吞吐量受限的部署场景中格外突出。与此相对，基于记忆层（Memory Layer）的稀疏架构通过嵌入检索机制实现稀疏激活，理论上能够以更低的内存访问代价完成等效计算，然而此前的代表性工作 UltraMem 仅能追平 2-expert MoE 的性能水平，与主流 8-expert MoE 相比存在显著差距。这一鸿沟极大地制约了记忆层技术的实际落地价值，成为该领域亟待解决的核心问题。

UltraMemV2 正是在这一背景下被提出的，其目标是通过系统性的架构重设计，弥合记忆层架构与先进 MoE 模型之间的性能差距。论文的核心方法建立在 Tucker 分解的查询键检索（Tucker Decomposed Query-Key Retrieval，TDQKR）机制之上，通过在每个 Transformer 块中统一集成记忆层（而非此前稀疏插入的方式），大幅提升了稀疏参数在整个模型计算流程中的参与密度。具体而言，UltraMemV2 引入了五项关键改进：第一，将记忆层覆盖到每一个 Transformer 块，而非仅在少数层中插入，从根本上提升了稀疏参数的利用率；第二，对隐式值扩展（Implicit Value Expansion）机制进行简化，采用单一线性投影替代多个每头线性层，在保留表达能力的同时减少非计算开销；第三，借鉴 PEER 模型的设计，将原先的简单值嵌入替换为带有单内维度的 FFN 计算单元，并去除激活函数以提升参数利用效率；第四，基于中心极限定理推导出原则性的参数初始化策略，通过匹配记忆层输出方差与 FFN 输出方差来防止训练过程中的梯度发散问题，其初始化标准差由公式

$$\sigma_V^2 = \sqrt{\frac{0.2 \times k_{\text{inner}} \times h}{k \times n_{\text{head}} \times (1 + \sigma_s^2) \times d_{\text{prev}} \times d_v \times L}}$$

确定，其中 $k$ 为激活值数量，$n_{\text{head}}$ 为注意力头数，$L$ 为层数；第五，通过系统性消融实验确定最优的记忆层与 FFN 计算比例，发现在 500M 和 6B 规模模型上，记忆层计算占比（Memory Computational Proportion）设定为 17% 时性能最优，对应键维度 $D_k = h/2$（即键维度等于隐层宽度的一半）。此外，研究还发现激活值密度（top-$m$ 激活数量）对性能的影响显著大于稀疏参数总量，即 top-768 from 60B 参数配置优于 top-256 from 120B 参数配置，尽管两者的浮点计算量完全相同，这一发现具有重要的架构设计指导意义。

在实验设计方面，论文在多个规模和数据量级上验证了 UltraMemV2 的有效性。主要模型包括：在内部专有数据上训练的 UltraMemV2-2.5B/120B-top256（激活参数 2.5B，稀疏参数总量 120B，每层激活 256 个值）和 UltraMemV2-2.5B/60B-top768（每层激活 768 个值），分别采用 1.6T tokens 预训练加 250B tokens 持续训练（CT），以及扩展版本 3.9T tokens 预训练加 500B tokens CT；此外还发布了开源版本 UltraMemV2-227M/1.2B 和 UltraMemV2-1B/7B，分别在 500B 或 1T tokens 上训练。主要对比基线为同等计算量和参数量的 SeedMoE-2.5B/30B（30B 总参数、8 个专家配置）。在通用能力评测方面，经过完整 3.9T+500B CT 训练后，UltraMemV2-2.5B/120B 在 OpenBench All 上得分 70.7，HardBench All 上得分 31.7，与 SeedMoE 基线（OpenBench 70.3，HardBench 30.3）实现性能持平，证明记忆层架构可以在与 8-expert MoE 相同计算预算下达到等效的通用语言理解能力。更引人注目的结果体现在记忆密集型任务上（Table 2）：在长上下文记忆化任务中，UltraMemV2 得分 23.5，优于 MoE 基线的 21.9，提升 +1.6 分；在多轮记忆化任务中，UltraMemV2 得分 31.2，优于 MoE 的 25.0，提升 +6.2 分；在上下文学习（in-context learning）任务中，UltraMemV2 得分 29.5，大幅优于 MoE 的 21.6，提升 +7.9 分，系统性地展示了记忆层架构在需要精确记忆和检索信息场景下的内在优势。消融实验进一步量化了各项改进的贡献：引入 PEER 风格的 FFN 值处理带来 0.7 分的准确率提升；单投影器加多查询配置同样贡献 0.7 分；单头配置额外带来 0.2 分提升；而在持续训练阶段，至少需要 20 层 UltraMem 层才能获得具有竞争力的增益。在开源模型对比中，UltraMemV2-1B/7B 经过 1T tokens 训练后在标准基准上达到与 OLMoE 相当的性能，验证了方法在更广泛开放评测环境下的适用性。

UltraMemV2 的研究意义在于从技术上证明了记忆层架构有能力作为 MoE 模型的竞争性替代方案，并在记忆密集型任务上展现出独特优势。在大规模语言模型普遍面临长上下文处理、多轮对话连贯性等挑战的背景下，UltraMemV2 提供了一条既能降低推理内存访问成本又能增强记忆能力的技术路径。论文揭示的"激活密度比总参数量更重要"这一规律，为未来稀疏架构的设计提供了明确的优化方向——与其单纯堆砌稀疏参数总量，不如优先提高每层激活值的密度。此外，五项系统性改进均有理论推导支撑（尤其是基于中心极限定理的初始化策略），使得该架构不仅在实践上有效，在理论上也具有较好的可解释性，为后续研究记忆层扩展规律和 MoE 架构演化的工作奠定了坚实基础。模型代码和权重的开源发布也将推动社区在此基础上开展更深入的探索与应用。
