# Nemotron-CC-Math: A 133 Billion-Token-Scale High Quality Math Pretraining Dataset

> **Venue**: ICLR 2026 | **ArXiv**: 2508.15096 | **NemotronCCMath**

数学推理能力是大语言模型（LLM）综合智能的核心组成部分，而预训练数据的质量与规模对最终模型性能起着决定性作用。现有的开源数学预训练数据集，如 OpenWebMath、FineMath 和 MegaMath，均从 Common Crawl 网页语料中提取数学内容，然而这些数据集普遍面临严重的质量问题：由于通用文本提取工具（如 Trafilatura、Resiliparse、jusText）是为叙述性文本设计的，无法正确处理数学公式的多样化编码格式（如 MathJax、KaTeX、MathML），导致数学符号在提取过程中频繁被截断或损坏，行间 LaTeX 公式被错误剥离从而改变语义，代码块的缩进结构也常常遭到破坏。这些质量缺陷直接制约了下游模型的数学与推理能力上限。NVIDIA 团队正是在这一背景下提出了 Nemotron-CC-Math，通过重新设计整条数据提取与清洗流水线，构建了目前规模最大、质量最高的开源数学预训练数据集。

Nemotron-CC-Math 的核心技术创新在于其五阶段数据处理流水线。**第一阶段**采用基于 Lynx 文本浏览器的布局感知渲染方案替代传统的 DOM 解析方式。Lynx 作为一款命令行文本浏览器，能够将网页渲染为保留数学公式和代码格式的纯文本，其布局感知特性确保了数学内容在格式转换过程中的结构完整性。相较于现有工具，Lynx 渲染方案在数学符号保真度方面具有显著优势，能够正确处理多种数学渲染框架输出的内容。**第二阶段**引入 Phi-4（14B 参数）大语言模型对 Lynx 输出进行语义清洗：移除页眉页脚、导航栏等样板内容；将 MathML、KaTeX、MathJax 等多种格式统一标准化为 LaTeX 表示；纠正排版错误并补全不完整的公式。值得注意的是，研究者对多个规模的 LLM（包括 DeepSeek-V3 671B、Qwen2.5-72B、Qwen2.5-32B 和 Phi-4 14B）进行了对比消融，发现体量仅为最大模型 1/47 的 Phi-4 在清洗质量上与更大模型旗鼓相当甚至更优，这表明样板内容去除任务本身并不需要超大规模模型。**第三阶段**使用基于 FineMath 五分制量表的质量分类器对文档进行打分，该量表聚焦于适合中学及高中水平的数学推理内容。**第四阶段**采用基于 MinHash 的局部敏感哈希（LSH）算法进行模糊去重，以 24 字符 n-gram 和 Jaccard 相似度阈值 0.8 识别近重复文档。**第五阶段**进行基准污染检测，通过基于嵌入的余弦相似度（阈值 0.9）将与评测集提示或答案高度相似的文档从训练数据中剔除，实际去除比例不足 0.002%，但这一步骤对评测结果的可靠性至关重要。

在数据规模与构成方面，流水线处理了来自 98 个 Common Crawl 快照（2014—2024 年）的 2.295 亿个网页，URL 集合来自 OpenWebMath、FineMath、MegaMath 和 InfiMM-WebMath 的预筛选结果。最终产出两个版本的数据集：**Nemotron-CC-Math-4+**（质量分 4-5，52.32B tokens，4500 万篇文档）和 **Nemotron-CC-Math-3+**（质量分 3-5，133.26B tokens，1.01 亿篇文档）。Nemotron-CC-Math-4+ 包含的 token 数量是此前质量最高的开源数学数据集 FineMath-4+ 的 5.5 倍，在质量与规模上均实现了显著突破。数据来源以数学论坛和问答社区为主，mathhelpforum.com（8.44%）、jiskha.com（5.26%）、physicsforums.com（2.78%）和 math.stackexchange.com（2.35%）贡献了最多内容。学科分布约 76% 为数学，其余涵盖物理、统计、化学、经济学和计算机科学。此外，由于流水线在保留数学公式结构的同时意外保留了代码块格式，Nemotron-CC-Math-3+ 和 -4+ 分别包含约 430 万和 144 万条含代码片段的文档，这一附带收益为后续代码能力提升埋下伏笔。

实验设计以 Nemotron-T 8B 模型为基础，通过退火（annealing）消融实验评估不同数据集对预训练效果的影响。研究者设置了两种计算预算：**100B token** 消融用于评估规模较小的高质量数据集（< 30B tokens），**300B token** 消融用于比较更大规模数据集。在数据混合比例上，目标数学数据集占比 30%，其余比例按比例从基准混合中下采样。评测覆盖知识（MMLU、MMLU-Pro、MMLU-STEM）、数学推理（MATH、GSM8K）和代码（MBPP+、MBPP、HumanEval+、HumanEval）三大维度。在 100B token 消融中，与 FineMath-4+ 相比，Nemotron-CC-Math-4+ 在 MATH 上提升 +4.8 分（40.60 vs 35.80），MBPP+ 上提升 +16.23 分（45.11 vs 28.88）；与 MegaMath-Pro 相比，MATH 提升 +6.6 分，MMLU-Pro 提升 +2.1 分。在 300B token 消融中，与 FineMath-3+ 相比，Nemotron-CC-Math-3+ 在 MATH 上提升 +9.6 分（44.20 vs 34.60），MBPP+ 上提升 +14.32 分（43.51 vs 29.19）；与 MegaMath-Web 相比，MATH 提升高达 +12.6 分（44.20 vs 31.60）。在通用能力方面，MMLU-STEM 相较最强基线提升 +1.97—+5.06 分，表明高质量数学预训练对通用推理能力具有正迁移效果。对比实验同时验证了数学领域的收益不以牺牲其他能力为代价，呈现出全面均衡的性能提升。

Nemotron-CC-Math 的研究意义体现在多个层面。从数据工程角度，该工作首次系统性地论证了布局感知 HTML 渲染与 LLM 语义清洗相结合的必要性，为未来构建领域专用预训练数据集提供了可复用的方法论框架——其流水线已通过 NeMo-Curator 完整开源，具有很强的工程可迁移性。从规模与质量的权衡角度，Nemotron-CC-Math-4+ 以 5.5 倍于 FineMath-4+ 的 token 量实现了质量的同步提升，打破了"更大规模意味着更低质量"的传统认知，表明在合理的清洗流程下规模扩展与质量提升可以并行不悖。从开放科学角度，该工作承诺开放代码与数据集，有望推动社区在数学推理、科学问答等方向的进一步探索。此外，研究揭示的 Phi-4 小模型在清洗任务上媲美超大模型的发现，对数据工程中的成本效益优化具有重要启示：并非所有 LLM 辅助任务都需要部署最大规模的模型，合理的任务匹配可以大幅降低数据构建成本。总体而言，Nemotron-CC-Math 通过技术创新与规模扩展的有机结合，为大语言模型数学能力的预训练阶段提升树立了新的基准，其贡献涵盖数据集本身、方法论框架和工程工具三个维度，对 LLM 预训练研究社区具有重要的参考价值。
