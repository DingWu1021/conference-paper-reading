# LeRobot: An Open-Source Library for End-to-End Robot Learning

> **Venue**: ICLR 2026 | **ArXiv**: 2602.22818 | **LeRobot**

机器人学习领域长期面临一个根本性的工程困境：生态系统高度碎片化。研究者和工程师往往需要同时应对来自不同来源、互不兼容的中间件、数据格式与算法实现，这使得跨实验室的复现极为困难，也极大提高了新入门者的学习门槛。传统机器人系统依赖大量手工设计的模块化流水线，每一个子组件都需要领域专家精心调整，既难以迁移到新场景，也无法随着数据量和算力的增加而系统性地提升性能。与此同时，随着模仿学习、强化学习以及基于大模型的视觉-语言-动作（VLA）策略的兴起，端到端机器人学习逐渐展示出从原始感知直接映射到低层动作的强大潜力，这进一步凸显了一个统一、开放、可扩展的机器人学习框架的迫切需求。

本文提出的 LeRobot 正是为了解决上述问题而构建的一个开源端到端机器人学习库，由 Hugging Face 团队主导开发。LeRobot 的核心设计哲学围绕三个关键词展开：可及性（Accessibility）、可扩展性（Scalability）与开放性（Openness）。在可及性方面，LeRobot 支持以极低成本搭建机器人数据采集系统——例如 SO-100/SO-101 机械臂的硬件成本约为 225 欧元，ALOHA-2 约为 21000 欧元，远低于传统工业级机器人的价格。库本身采用纯 Python 实现，训练一个策略所需代码不超过 100 行，部署推理不超过 40 行，极大降低了入门门槛。在可扩展性方面，LeRobot 覆盖了从低层电机通信到大规模数据集管理再到前沿算法实现的完整技术栈，支持流式数据加载以应对百万量级的演示数据集，并通过异步推理栈实现高效在线部署。在开放性方面，LeRobot 的全部代码、数据集与预训练权重均对外公开，截至 2025 年 9 月，Hugging Face Hub 上已有超过 2200 位贡献者上传了超过 16000 个机器人学习数据集。

从技术架构来看，LeRobot 的技术贡献可归纳为四个主要模块。第一是统一的机器人中间件 API。LeRobot 为八种主流机器人平台提供了统一的 Python 接口，包括 SO-100/101 操纵臂、Koch-v1.1、ALOHA-2、Hope-JR 仿人手臂、Stretch-3 移动操纵臂、LeKiwi 移动机器人以及 Reachy-2 仿人机器人。该 API 直接对接 FeeTech 和 Dynamixel 等主流执行器厂商的驱动，并采用领导者-跟随者（Leader-Follower）架构同时支持遥操作数据采集与策略控制部署，使同一套代码能在采集和推理两种模式下无缝切换。第二是标准化数据集格式 LeRobotDataset。传统机器人数据集散落于 TensorFlow Datasets、ROS bag、自定义 JSON 等五花八门的格式中，LeRobotDataset 通过将表格型数据存储为高效的 Parquet 文件、将视频帧压缩为 MP4 格式，并附加包含任务描述、机器人本体规格、采集频率、传感器类型等信息的自包含元数据，实现了跨平台、跨实验室的数据标准化。更重要的是，LeRobotDataset 支持按需从远端存储流式读取帧数据，无需将整个数据集预加载到本地，使得处理包含数十万乃至数百万条演示轨迹的大规模数据集成为可能。第三是多范式算法支持。LeRobot 在单一代码库中实现了多种前沿机器人学习算法，覆盖模仿学习与强化学习两大范式。在模仿学习方面，支持动作分块 Transformer（ACT，52M 参数）、扩散策略（Diffusion Policy，263M 参数）、向量量化行为 Transformer（VQ-BET）、$\pi_0$（35 亿参数的视觉-语言-动作流模型）以及 SmolVLA（450M 参数，专为资源受限部署设计）。在强化学习方面，支持 HIL-SERL（人在环强化学习）和 TD-MPC（时序差分模型预测控制），其中 HIL-SERL 能够在 1 至 2 小时的训练内在复杂操纵任务上达到近乎完美的成功率。第四是广义异步推理栈。LeRobot 的推理栈实现了物理解耦（Physical Decoupling）与逻辑解耦（Logical Decoupling）的双重分离。物理解耦允许将计算密集型的神经网络推理迁移到远端高性能 GPU 服务器上运行，而机器人控制循环则在本地硬件上以所需频率执行，两者通过网络连接协同工作。逻辑解耦则采用异步生产者-消费者模式：推理线程以前瞻时域 $H$ 预测动作序列，并将其放入动作队列；控制线程按固定频率从队列中取出并执行动作，同时推理线程已在后台预测下一段动作，两者并行运行，通过可定制的聚合函数 $f$ 无缝衔接动作序列之间的过渡。

在实验评估方面，论文从推理效率与实机操纵任务两个维度对 LeRobot 进行了系统性验证。推理延迟基准测试（100 次前向传播的平均耗时，以毫秒为单位）揭示了各模型对计算资源的不同依赖程度：ACT 在 CPU 上耗时 182.3ms，在 RTX 4090 上仅需 5.0ms；Diffusion Policy 在 CPU 上高达 3453.8ms，在 RTX 4090 上为 613.9ms；$\pi_0$ 在 CPU 上已超时，在 A100 上需 569.0ms；SmolVLA 在 RTX 4090 上耗时 99.2ms。显存占用方面（fp32 精度），ACT 为 817.4MB，Diffusion Policy 为 1.22GB，$\pi_0$ 为 4.13GB，SmolVLA 为 1.69GB。实机任务评估在 SO-100 机器人上测试了捡放（Pick-Place）、堆叠（Stacking）、分拣（Sorting）三项任务，对比同步与异步两种推理模式：捡放任务同步成功率 75%、异步 80%；堆叠任务两种模式均达 90%；分拣任务同步 70%、异步 50%，显示异步模式并非在所有任务上都优于同步模式，其优势主要体现在吞吐量层面——在固定 60 秒的操纵窗口内，异步模式可搬运 19 个积木，同步模式仅搬运 9 个，并将平均单任务时间从 13.75 秒缩短至 9.70 秒，提升约 29%。仿真评估则在 LIBERO（四个固定任务套件，涵盖空间、物体、目标等变体）和 Meta-World（50 个操纵任务，支持多任务与元学习评估）两个标准基准上进行，验证了各算法实现的正确性与互操作性。

LeRobot 的学术意义和实践价值体现在多个层面。从研究基础设施角度看，LeRobot 提供了一个真正意义上覆盖"从电机到模型"完整技术栈的统一平台，使得端到端可复现的机器人学习实验成为可能，有望成为机器人学习领域的事实标准开发框架，类似于 Hugging Face Transformers 之于自然语言处理领域的地位。从技术趋势角度看，LeRobot 明确拥抱"以学习代替手工"的范式转移，将强化学习、模仿学习与大型视觉-语言-动作模型统一在同一框架下，并通过异步推理栈解决了大模型部署在机器人实时控制场景下的延迟瓶颈问题。从社区生态角度看，截至 2025 年 9 月超过 16000 个开放数据集的存在表明，LeRobot 已成功催生了一个活跃的去中心化数据采集社区，这种社区驱动的数据积累模式对于训练具有广泛泛化能力的通用机器人策略至关重要。当然，LeRobot 目前仍存在若干局限性：支持的机器人平台数量有限（尽管已从最初的 3 种扩展到 8 种）、算法覆盖并不穷举、尚未支持量化和图编译等低层次推理优化手段。作者将这些开放问题明确列为未来社区贡献的方向，体现了开源协作的精神。总体而言，LeRobot 代表了机器人学习领域在基础设施层面的一次重要统一尝试，其长远影响将随着社区贡献的积累而持续放大。
