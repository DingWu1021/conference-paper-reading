# FSA: An Alternative Efficient Implementation of Native Sparse Attention Kernel

> **Venue**: ICLR 2026 | **ArXiv**: 2508.18224 | **FSASparseAttn**

随着大语言模型（LLM）在长上下文场景中的广泛应用，注意力计算的效率瓶颈愈发凸显。研究表明，在 64K 词元的上下文长度下，注意力机制可占解码总延迟的 70%～80%。为缓解这一问题，稀疏注意力方法应运而生，其中 Native Sparse Attention（NSA）因在算法层面实现了显著的计算量削减而受到广泛关注。然而，NSA 的内核（kernel）实现存在一个关键的设计约束：其循环结构要求每个分组查询注意力（Grouped Query Attention，GQA）组中包含较多的查询头（query head）才能充分发挥硬件加速效率。具体而言，NSA 采用"先遍历查询词元、再遍历 KV 块"的外-内双层循环结构，每个线程块将同一 KV 头对应的所有查询头批量处理。当 GQA 组内的查询头数量较少（如小于 8）时，为满足 GPU 矩阵乘法对最小维度的硬件要求，NSA 不得不对查询头进行填充（padding），由此引入了大量无效内存访问与冗余计算，严重影响实际性能。然而，当前主流 LLM（如 Llama3-8B、Qwen3-14B、Qwen2.5-32B 等）普遍采用较小的 GQA 组大小，这使得 NSA 在实际部署中的加速效果大打折扣，理论效率难以转化为真实收益。本文正是针对这一理论与工程之间的鸿沟，提出了 Flash Sparse Attention（FSA）作为 NSA 的一种替代内核实现方案。

FSA 的核心创新在于对 NSA 双层循环顺序的反转。FSA 将循环结构调整为"先遍历 KV 块、再遍历查询词元"，即以 KV 块为外循环、以查询词元为内循环。这一看似简单的调整带来了根本性的优势：每个线程块负责处理所有关注同一 KV 块的查询词元，无需对查询头进行填充，从而彻底消除了 GQA 组大小较小时的冗余开销。以典型配置（KV 块大小 $B_K=64$，查询批大小 $T=16$，序列长度 64K）为例，当 GQA 组大小为 4 时，FSA 的内存访问量仅为 NSA 的 21.3%，浮点运算量仅为 NSA 的 56.2%，且随着 GQA 组大小的进一步减小或 KV 块大小的增大，这一优势还会继续放大。此外，FSA 的因果掩码（causal masking）可以自然满足，无需额外的掩码操作，进一步节省了计算资源。

然而，循环顺序的反转也带来了新的工程挑战，主要体现在在线 softmax 的管理与非连续内存访问的优化两个方面。传统 FlashAttention 风格的在线 softmax 依赖于对 KV 块的顺序扫描来维护数值稳定的归一化统计量，而 FSA 的并行化方式使得不同 KV 块的计算可以独立进行，无法直接复用传统方案。为此，FSA 设计了一套三阶段解耦方案：首先，预计算内核（pre-computation kernel）利用查询张量和键张量提前计算在线 softmax 统计量；随后，选择注意力内核（selected attention kernel）在应用这些统计量的同时完成主体计算；最后，归约内核（reduction kernel）对各 KV 块的部分结果进行正确缩放并合并，得到最终输出。对于非连续内存访问问题，FSA 引入了索引张量（index tensor）机制，并配合提前返回（early return）策略，在加载每批查询词元之前先检查其是否确实关注当前 KV 块，从而过滤掉不必要的内存加载，有效降低了非连续访问的开销。在硬件资源分配上，FSA 采用细粒度的 warp 分配策略，将 4 个 warp 用于矩阵运算，1～2 个 warp 用于归约操作，并通过输出缓冲区（$O_{\text{buf}}$）的复用与紧凑索引映射，将 64K 词元场景下约 11 GB 的缓冲区开销控制在现代 GPU（显存容量通常为 96～141 GB）的可接受范围之内。整个实现包含约 10,000 行 Python/Triton 代码，充分体现了算法与系统协同设计的复杂性。

在实验评估方面，作者在多款主流 LLM 上对 FSA 进行了全面测试，涵盖内核级性能、端到端训练加速和推理预填充（prefill）加速三个维度。内核级延迟测试表明，与原始 NSA 相比，FSA 最高可实现 3.5 倍的加速，平均加速比为 1.6 倍；与全注意力（full attention）相比，在 H20 GPU 上最高可达 6.4 倍加速。在端到端训练场景中，分别在 Llama3-8B、Qwen3-14B 和 Qwen2.5-32B 上进行评估，FSA 相对于 NSA 最高取得 1.25 倍的训练吞吐提升，平均提升幅度为 1.09 倍。在推理预填充阶段，FSA 相对于 NSA 最高实现 1.36 倍加速，平均为 1.11 倍；相对于全注意力最高可达 1.69 倍加速。尤为值得关注的是反向传播（backward pass）的性能：由于 FSA 在反向传播中同样避免了 NSA 的索引张量计算开销，其加速效果更为突出，最高可达 NSA 的 4.32 倍。消融实验进一步验证了各设计组件的贡献：禁用内循环优化会导致平均性能下降 11.9%，禁用提前返回机制则导致平均性能下降 18.2%，说明两者均为整体性能不可或缺的组成部分。正确性验证方面，作者在 Llama3-8B 的训练过程中对比了 FSA、NSA 和全注意力的损失曲线，结果显示三者的收敛行为高度一致，证明了 FSA 在保持算法等价性的同时实现了工程层面的显著优化。

综合来看，FSA 的工作具有重要的研究意义与实用价值。它揭示了一个普遍存在但往往被忽视的问题：算法层面的效率提升未必能直接转化为系统层面的加速收益，两者之间存在内核设计与硬件特性适配的复杂鸿沟。FSA 通过循环顺序反转这一简洁而有效的思路，结合针对非连续内存访问、在线 softmax 分解和缓冲区管理的一系列精细工程优化，成功地将 NSA 的稀疏注意力算法优势扩展至当前主流 LLM 所采用的小 GQA 组配置，使得稀疏注意力的实用性大幅提升。这项工作不仅为长上下文 LLM 的高效训练与推理提供了一个即插即用的工程解决方案，更为未来稀疏注意力内核的设计提供了重要的方法论参考——在追求算法创新的同时，必须充分考虑目标硬件架构与实际模型配置的约束，方能真正实现"算法-系统协同设计"（algorithm-system co-design）的最大化收益。FSA 的代码已在 GitHub（Relaxed-System-Lab/Flash-Sparse-Attention）开源，有望成为稀疏长上下文推理与训练基础设施的重要组成部分。
