# The Art of Scaling Reinforcement Learning Compute for LLMs

> **Venue**: ICLR 2026 | **ArXiv**: 2510.13786 | **ScalingRLLLM**

强化学习（Reinforcement Learning，RL）已经成为训练大型语言模型（LLM）不可或缺的核心环节，尤其是在提升模型推理能力方面发挥了关键作用。然而，与预训练领域已建立起完善的幂律缩放定律（power-law scaling law）体系不同，RL 训练至今缺乏可预测的扩展方法论。研究者们在面对不断攀升的计算预算时，往往依赖直觉和经验性调参，无法在小规模实验中可靠地预测大规模训练的效果。这种"艺术性"而非"科学性"的现状，使得 RL 训练的资源投入存在极大的不确定性和浪费。与此同时，预训练领域的 Chinchilla 等工作已经为模型规模与数据量的最优分配提供了严格的理论指导，而 RL 后训练阶段的规模化方法论探索依然是一片空白。本文正是在这一背景下，系统性地展开了迄今为止规模最大的 RL 扩展研究，累计消耗超过 40 万 GPU 小时的算力，旨在将 RL 训练从"艺术"推进为"科学"。

在方法论层面，本文的核心贡献是构建了一套基于 S 型（sigmoidal）计算-性能曲线的分析框架。作者观察到，在以对数坐标轴表示计算量时，RL 训练的通过率（pass rate）随计算量的变化呈现出稳定的 S 型曲线，其形式为：

$$R_C - R_0 = (A - R_0) \times \frac{1}{1 + (C_{\mathrm{mid}} / C)^B}$$

其中 $A$ 表示渐近性能上限（asymptotic performance ceiling），$B$ 表示计算效率（斜率参数），$C_{\mathrm{mid}}$ 为曲线拐点处的计算量，$R_0$ 为初始性能基线。这一框架比幂律拟合更为稳健，能够在小规模运行中可靠地拟合并外推到更大计算预算下的性能。基于此框架，作者提出了三条核心原则：第一，不同的训练方案会导致不同的渐近性能上限 $A$，并非所有算法最终都能达到相同的天花板；第二，"苦涩的教训"在 RL 中同样适用——在小计算量下表现优异的方法，不一定在大计算量下具备优势，必须依据拟合参数 $(A, B)$ 而非早期表现来评估算法；第三，众多设计选择（如损失聚合方式、归一化策略、课程学习、异步训练等）主要影响的是计算效率参数 $B$，而非性能上限 $A$，二者的分离分析是理解 RL 扩展行为的关键。

在实验设计方面，本文基于一个 8B 参数的稠密语言模型展开研究，训练域为数学推理（利用可验证奖励信号进行 RL），序列长度设定为 16384 个 token，批量大小为 768（48 个提示 × 每提示 16 次采样），验证集包含 1000 个保留提示。基础设施采用生成器-训练器分离架构，结合异步流式采样，运行于 Nvidia GB200 集群。消融实验覆盖了业界最常见的设计选择：损失函数类型（DAPO、GSPO、CISPO 等）、重要性采样比值的数值精度（FP32 vs BF16）、异步 off-policy 设置（PipelineRL vs PPO-off-policy-k）、损失聚合粒度（token 级、样本级、提示级）、优势归一化范围（batch 级 vs mini-batch 级）、零方差样本过滤，以及自适应提示过滤（No-Positive-Resampling）。

在主要实验结果方面，消融研究揭示了若干关键结论。在损失函数选取上，CISPO 和 GSPO 相比 DAPO 能够将渐近性能从 $A = 0.52$ 提升至 $A = 0.61$，效果显著；而 CISPO 在长计算量阶段表现出近似线性的缓慢增长特性，略优于 GSPO。在数值精度方面，在语言模型 head 处强制使用 FP32 精度计算对数概率，对于避免重要性采样比值的数值不稳定至关重要，同样能将渐近性能提升约 0.09 个绝对百分点。在异步训练方面，PipelineRL-8（8 步 off-policyness）比 PPO-off-policy-k 在相同渐近性能下具有更快的收敛速度，主要得益于其更少的 GPU 空闲时间。在课程学习上，No-Positive-Resampling（过滤通过率 $\geq 0.9$ 的提示）在大计算量下能有效提升扩展性，而零方差过滤（排除奖励方差为零的批次）也能小幅改善渐近上限。综合以上所有发现，作者提出了 ScaleRL 方案，其正式目标函数为：

$$J_{\mathrm{ScaleRL}}(\theta) = \mathbb{E}\left[\frac{1}{\sum_g |y_g|} \sum_i \sum_t \mathrm{sg}\!\left(\min(\rho_{i,t}, \varepsilon)\right) \hat{A}_i^{\mathrm{norm}} \log \pi_{\mathrm{train}}(y_{i,t})\right]$$

该方案将 PipelineRL（8 步 off-policyness）、FP32 logit 计算、CISPO 损失、提示级损失聚合、batch 级优势归一化、零方差过滤、No-Positive-Resampling 课程以及强制长度中断控制有机结合为一个完整训练框架。在与业界主流方案的横向对比中，ScaleRL（$A = 0.61$）优于 DeepSeek GRPO（$A = 0.57$）、Qwen-2.5 DAPO（$A = 0.59$）、Magistral（$A = 0.60$）及 MiniMax（$A = 0.60$）等方法。最重要的验证实验表明，用 ScaleRL 在 5 万 GPU 小时的小规模运行中拟合得到的 S 型曲线，成功外推并预测了 10 万 GPU 小时下的真实性能，二者高度吻合（误差在 $\pm 0.02$ 范围内），充分证明了该框架的预测能力。

在多轴扩展研究（Section 5）方面，作者还系统探索了模型规模、生成长度、批量大小、每提示生成数等多个维度。17B×16 的 MoE 模型仅需 8B 稠密模型约六分之一的 RL 计算量，即可达到更高的渐近性能，说明更大的预训练模型本身即可带来 RL 效率的显著提升。将生成长度从 14K token 扩展至 32K token，虽然初期收敛较慢，但最终能达到更高的性能上限，属于"提升天花板"的策略而非纯粹的效率取舍。更大的批量大小（batch=2048）虽在早期出现短暂停滞，但最终同样对应更高的渐近性能，并在 10 万 GPU 小时的长期训练中表现出更好的稳定性。上述各维度的扩展收益均与下游基准（如 AIME-24）的提升存在一致的正相关性，验证了分布内扩展规律具有良好的泛化性。

本文的研究意义是深远的。它首次为 RL 后训练阶段建立了类似预训练 Chinchilla 定律的预测性框架，将过去依赖直觉的"调参艺术"转化为可量化、可外推的科学方法。通过系统区分影响渐近上限 $A$ 与影响计算效率 $B$ 的设计因素，研究者和工程师可以在早期小规模实验中快速筛选出真正提升性能天花板的关键设计选择，从而大幅降低超大规模 RL 训练的试错成本。ScaleRL 方案本身也提供了一个经过严格验证的最佳实践基线，可供社区在此基础上继续探索。更广义地看，本文所建立的"拟合 S 型曲线 → 分析参数 → 外推预测"的分析范式，有潜力被推广到代码生成、多模态推理等其他 RL 训练场景，为整个大模型 RL 训练体系的科学化奠定了重要基础。
