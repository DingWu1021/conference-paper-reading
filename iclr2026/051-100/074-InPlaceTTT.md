# In-Place Test-Time Training

> **Venue**: ICLR 2026 Oral | **ArXiv**: 2604.06169 | **InPlaceTTT**

大型语言模型（LLM）的主流开发范式是"先训练、后部署"——模型一旦完成预训练，其权重便被冻结，在推理阶段不再更新。这种静态范式在面对真实世界中持续涌现的新信息时显现出根本性的局限：模型无法根据当前输入上下文动态调整自身表示，从而难以在超长上下文、持续学习等场景中保持高质量的推断能力。虽然上下文学习（In-Context Learning）提供了一定程度的灵活性，但其依赖注意力机制的二次方复杂度，使得处理数十万乃至更长的序列在计算上代价高昂。测试时训练（Test-Time Training，TTT）作为一种在推理阶段利用当前输入对模型权重进行动态更新的思路，理论上能弥补上述短板，然而将 TTT 真正引入当代 LLM 生态系统面临三大关键障碍：其一，现有 TTT 方法通常需要引入专用的新型层结构，与现有 Transformer 架构存在不兼容性，从零预训练的成本极高；其二，逐 token 的顺序更新固有地破坏了现代 GPU/TPU 对并行计算的优化，带来严重的计算效率瓶颈；其三，现有 TTT 方法普遍采用重建（Reconstruction）目标，即令模型记忆当前 token 本身，这与自回归语言建模所追求的"预测下一个 token"目标存在根本性错位。正是为了系统性地解决上述三大障碍，本文提出了 In-Place Test-Time Training（In-Place TTT）框架。

In-Place TTT 的核心思路是将现有 Transformer 中无处不在的 MLP 块的最终投影矩阵重新诠释为可在推理时动态更新的"快权重"（fast weights），从而实现对已有模型的"即插即用"式增强，而无需从零重新设计架构。具体而言，在标准门控 MLP 结构中，输出计算为 $O = (\phi(H W_{\text{gate}}^T) \odot (H W_{\text{up}}^T)) W_{\text{down}}^T$，In-Place TTT 将 $W_{\text{down}}$ 视为可适应的快权重，而保持 $W_{\text{up}}$ 与 $W_{\text{gate}}$ 不变。这种设计以最小的侵入性保留了预训练权重的知识，同时赋予模型在推理时动态更新自身的能力。为了克服逐 token 顺序更新的效率瓶颈，框架采用"块状更新"（chunk-wise update）策略：将序列划分为大小为 $C$ 的不重叠块，对每个块执行并行的"应用"与"更新"操作。应用操作为 $O_{[i]} = Z_{[i]} (W_{\text{down}}^{(i)})^T$，更新操作为 $W_{\text{down}}^{(i+1)} = W_{\text{down}}^{(i)} - \eta \nabla_W \mathcal{L}(Z_{[i]} (W_{\text{down}}^{(i)})^T, V_{[i]})$。当块大小 $C$ 取 512 或 1024 时，可在现代加速器上实现高效的并行处理。更进一步，框架利用更新操作的关联性，通过三阶段并行化实现与上下文并行（Context Parallelism）的兼容：首先并行计算各块的中间激活 $Z_{[i]}$ 和增量 $\Delta W_{\text{down}}^{(i)}$，然后执行前缀和以累积更新量 $\Delta S_i = \sum_{j=1}^{i-1} \Delta W_j$，最后并行地将聚合更新应用于各块——在严格保持因果语义的同时充分发挥硬件并行优势。

In-Place TTT 最具理论创新性的贡献在于提出了与语言建模目标对齐的新型快权重更新目标。有别于将值目标（value target）设置为当前 token 的重建目标，本文设计了包含未来 token 信息的 LM 对齐目标：$\hat{V} = \text{Conv1D}(X_0) \cdot W_{\text{target}}$，其中 $X_0$ 为 token 嵌入，$W_{\text{target}}$ 为可学习投影，一维卷积算子负责控制融入多少未来 token 的信息。论文从理论层面通过"归纳头"（Induction Head）场景证明了这一设计的必要性：**定理1** 表明，当关键-值对 $(k^*, v^*)$ 首次出现于位置 $t^*$、键 $k^*$ 在位置 $n > t^*$ 再次出现时，LM 对齐目标使得正确下一 token 的 logit 期望变化满足 $\mathbb{E}[\Delta \ell_n[v^*]] \geq \lambda_{\text{lr}} \cdot c_{\text{norm}}^2 \cdot c_{\text{align}} > 0$，即快权重更新确实提升了正确预测的概率；而重建目标对应的期望变化量级仅为 $O(\varepsilon)$，几乎可以忽略不计。这从理论上清晰地刻画了两种目标在自回归语言建模任务上的本质差异。在实现细节上，利用相似度损失函数，梯度可以化简为闭合形式：$W_{\text{down}}^{(i)} = W_{\text{down}}^{(i-1)} + \eta \hat{V}_{[i]}^T Z_{[i]}$，使得更新计算既高效又直观。

在实验验证方面，论文从两个维度对 In-Place TTT 进行了全面评估。第一维度是对已有 LLM 的"即插即用"增强实验：以 Qwen3-4B-Base 为基础模型，通过分两阶段的持续预训练（第一阶段约 200 亿 token、32k 上下文长度，第二阶段约 150 亿 token、128k 上下文长度）引入 In-Place TTT 能力，在 RULER 长上下文基准上进行评测。结果显示，In-Place TTT 在所有上下文长度下均带来一致性提升，且提升幅度随上下文长度增加而增大：在 128k 上下文下绝对提升达 2.2 个百分点，在超出训练范围的 256k 上下文外推场景中同样保持领先（43.9% 对比基线 41.7%）。该方法还成功泛化至 LLaMA-3.1-8B 和 Qwen3-14B-Base，验证了其跨模型通用性。第二维度是从零预训练的对比实验：在 500M、1.5B、4B 三种参数规模下，与滑动窗口注意力（SWA）、门控线性注意力（GLA）、DeltaNet、大块 TTT（LaCT）等竞争性基线进行系统对比。In-Place TTT 在所有规模的滑动窗口困惑度指标上均取得最低值，在 4B 规模的下游任务（HellaSwag、ARC、MMLU、PIQA）上表现优越，在 RULER 长上下文基准的 16k 场景下相对于全注意力基线实现了 6.58 → 19.99 的巨幅提升（+203%）。消融实验进一步验证了各设计要素的贡献：TTT 启用层数的增加带来单调性性能提升；块大小 $C=512$ 或 $C=1024$ 时性能最优，过小的块则会损害效果；LM 对齐目标中的 Conv1D 算子对长上下文性能至关重要，而 $W_{\text{target}}$ 投影对短上下文任务不可或缺，二者缺一不可；吞吐量与显存开销分析证实了额外引入的计算负担可忽略不计，框架在实际部署中具备高度可行性。

综合而言，In-Place TTT 从架构兼容性、计算效率、目标对齐三个维度系统性地攻克了将测试时训练引入现代 LLM 的核心障碍，提出了一套既有理论保证又有工程可行性的解决方案。该工作的意义不仅在于提升了 LLM 在长上下文场景下的表现，更在于开创了一条让预训练模型能够在推理阶段持续自适应的实用路径——这对于知识密集型任务、动态信息处理以及未来持续学习系统的构建均具有深远的参考价值。代码已开源于 ByteDance-Seed/In-Place-TTT。
