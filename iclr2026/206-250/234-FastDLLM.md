# Fast-dLLM: Training-free Acceleration of Diffusion LLM by Enabling KV Cache and Parallel Decoding

> **Venue**: ICLR 2026 | **ArXiv**: 2505.22618 | **FastDLLM**

扩散式大语言模型（Diffusion LLMs）近年来因其天然支持非自回归并行解码的特性而受到广泛关注，代表性工作包括 LLaDA、Dream 等。与传统自回归模型逐 token 生成不同，扩散语言模型通过迭代去噪过程同时预测多个 token，理论上具备更高的推理并行度。然而，在实际部署中，这类模型的推理吞吐量却普遍落后于以 LLaMA 为代表的自回归模型。究其根本原因，有两个关键瓶颈：其一，扩散语言模型采用双向注意力机制，现有的 KV Cache 技术是专为单向因果注意力设计的，无法直接适用于双向注意力，导致每个解码步骤都需要对全部 token 重新计算注意力，计算冗余极大；其二，当模型尝试在单步内同时解码多个 token 时，其质量会出现显著下降，因为同时解码违背了 token 间的真实条件依赖关系，迫使模型在条件独立假设下采样，产生所谓"并行解码诅咒"。这两大障碍共同制约了扩散语言模型在实际应用中的推理效率。

Fast-dLLM 针对上述两个瓶颈，分别提出了无需训练的解决方案，从而在保持生成质量的前提下大幅提升推理吞吐量。

**块级近似 KV Cache（Block-wise Approximate KV Cache）** 是本文应对双向注意力缓存问题的核心创新。研究者通过实验观察到一个重要经验规律：在扩散模型相邻的若干解码步骤中，注意力层的键值激活（KV activations）在余弦相似度上非常接近 1.0，表现出高度相似性。这一现象启示了"近似复用"的可行性——即在一个由若干解码步骤构成的"块"内，可以将第一步计算出的 KV 缓存复用于后续步骤，块结束后再统一更新缓存。基于此，论文提出两种变体：**PrefixCache** 仅缓存提示词（prompt）对应的 token 的 KV 激活，在每个解码步骤中只为生成区域重新计算注意力；**DualCache** 进一步将缓存扩展到前缀和后缀（含掩码 token）区域，实现更激进的加速。由于缓存更新可与解码步骤联合执行，相较于不使用缓存的基线，DualCache 不引入任何额外计算开销。实验表明，当提示词较长（如 8-shot 场景）或生成长度较大（如 1024 tokens）时，KV Cache 的加速效果会进一步放大，原因在于提示词 token 在总计算量中占据更大比例，缓存带来的节省也随之增加。

**置信度感知并行解码（Confidence-Aware Parallel Decoding）** 则是针对多 token 并行解码质量退化问题的精准解法。论文从理论上对"并行解码诅咒"进行了刻画：当模型对 $n$ 个 token 并行采样时，实际上是在用各 token 边缘分布之积 $\prod_i p(x_i|\cdot)$ 代替真实联合分布 $p(x_1, x_2, \ldots, x_n|\cdot)$，忽略了 token 间的互相依赖。为此，论文给出了 **定理 1**：当每个 token 的预测置信度均超过阈值 $(1-\varepsilon)$ 时，若满足 $\varepsilon \leq \frac{1}{n+1}$，则并行贪心解码与顺序贪心解码等价，且该界是紧的（tight）。这一理论结果为实践策略提供了严格保障。在此基础上，论文提出两种具体解码策略：**阈值策略（Threshold-Based）** 在每步仅解码置信度超过给定阈值（默认 0.9）的 token，置信度不足的 token 保持掩码状态留待后续步骤处理；**因子策略（Factor-Based）** 则自适应地选取满足 $(n+1)(1-c^{(n)}) < f$ 的最大 $n$ 个 token 进行并行解码，其中 $c^{(n)}$ 为第 $n$ 高置信度，$f$ 为可调超参数，该策略在保证质量的同时能动态控制每步并行程度，进一步提升吞吐量。

在实验设计上，Fast-dLLM 选取了开源扩散语言模型中最具代表性的 LLaDA（7B）、LLaDA-1.5（7B）、Dream-Base 及多模态版本 LLaDA-V 作为评测对象，覆盖了数学推理（GSM8K、MATH）、代码生成（HumanEval、MBPP）和多模态理解（MathVista、MathVerse）等多类下游任务，硬件环境为单张 NVIDIA A100 80GB GPU，批大小为 1。

主要结果方面，Fast-dLLM 取得了显著的加速效果。在 LLaDA 模型的 GSM8K 任务中，生成长度为 256 tokens 时可实现 8.1 倍吞吐量提升，精度从 79.3% 仅微降至 78.5%；生成长度为 512 tokens 时吞吐量提升达 11.0 倍，精度几乎无损（77.5% 降至 77.2%）。在 MBPP 代码生成任务（生成长度 512）上，加速比达 9.2 倍，精度从 14.8% 降至 13.8%。在 8-shot 提示、生成长度 1024 的极端场景下，DualCache 可实现最高 **27.6 倍**的吞吐量提升，超越了此前所有针对扩散语言模型的加速方案，且与自回归模型 LLaMA 在小批量场景下的吞吐量持平乃至超越。在多模态扩散语言模型 LLaDA-V 上，Fast-dLLM 同样展现出 9.9 倍加速，MathVista 精度从 59.2% 微降至 56.6%，验证了方法的跨模态泛化能力。

消融实验进一步揭示了各设计选择的合理性：缓存块大小默认取 32 时在精度与吞吐之间取得最优平衡，过小的块会增加更新开销，过大的块则因上下文错配而损害精度；置信度阈值默认取 0.9，过低会引入质量退化，过高则限制了每步并行 token 数量；因子解码策略相比固定每步 token 数量的基线始终保持更高精度，印证了动态置信度感知的必要性。

总体而言，Fast-dLLM 是首个将 KV Cache 与置信度感知并行解码有机结合、系统性解决扩散语言模型推理效率问题的工作，且全程无需任何额外训练或微调。其理论上对并行解码等价条件的严格刻画（定理 1）和工程上对双向注意力缓存的巧妙近似，共同构成了本文的核心学术贡献。从更宏观的视角来看，扩散语言模型在长文本生成、受控生成和并行采样等方面具有自回归模型难以企及的天然优势，而推理效率长期以来是阻碍其大规模落地的主要障碍。Fast-dLLM 的工作有望在推理加速层面弥合扩散语言模型与自回归模型之间的性能鸿沟，为扩散语言模型的实际部署和后续研究奠定重要基础。
