# Zephyrus: An Agentic Framework for Weather Science

> **Venue**: ICLR 2026 | **ArXiv**: 2510.04017 | **ZephyrusWeather**

天气与气候预测长期以来依赖数值天气预报（Numerical Weather Prediction, NWP）系统，而近年来以深度学习为基础的气象基础模型（如 Pangu-Weather、GraphCast、FourCastNet 等）已在多项预测指标上超越了传统 NWP 方法，展现出强大的时空数据建模能力。然而，这些气象基础模型本质上处理的是高维结构化数值张量，无法直接与科研人员通过自然语言进行交互，更难以支撑需要多步推理、假设检验或反事实分析的复杂科学工作流程。与此同时，大语言模型（LLM）在自然语言理解与多步推理方面表现出色，但其训练数据中几乎不包含气象领域的高维时序数据，因此同样无法独立完成精细的气象分析任务。这一双重困境——气象模型不会"说话"，语言模型不懂"气象"——构成了推动 Zephyrus 工作的核心研究动机。来自加州大学圣地亚哥分校（UC San Diego）的研究团队（通讯作者 Sumanth Varambally 与 Rose Yu 等 17 位作者）为此提出了 Zephyrus：首个面向天气科学的智能体框架，旨在通过将 LLM 的语言推理能力与专业气象工具相结合，使研究人员能够以自然语言驱动复杂的气象分析任务。

Zephyrus 的核心技术贡献体现在三个紧密耦合的组件上。第一个组件是 **ZephyrusWorld**，一个为智能体量身设计的 Python 代码执行环境。该环境内置了一套统一的气象科学 API 工具集：其一是面向 WeatherBench 2 数据集的接口，提供了对 ERA5 再分析数据（覆盖 1979 年至 2022 年，空间分辨率 1.5°，时间分辨率 6 小时）的结构化检索能力；其二是基于 Natural Earth 地理数据集构建的双向地理编码工具（Geolocator），能够将自然语言中的地名描述转换为地理坐标或地理掩码，反之亦然；其三是集成了 Stormer 这一基于 Transformer 架构的神经天气预报模型的预报工具（Forecaster），支持最长两周的中短期天气预测；其四是基于 JAX-GCM 大气环流模型（融合 NeuralGCM 动力核心与 SPEEDY 物理参数化方案）构建的气候模拟工具（Simulator），可在约 25 秒内完成 5 天的大气模拟，极大地降低了气候推演的计算门槛。整个 ZephyrusWorld 后端采用 FastAPI 架构，通过资源池化机制管理并发代码执行，有效避免多任务之间的资源竞争问题。

第二个组件是 **Zephyrus 智能体**本身，系统提供了两种变体。Zephyrus-Direct 采用单步代码生成策略，智能体一次性生成完整的 Python 分析脚本，并在出现错误时进行最多 5 次自动修正迭代。Zephyrus-Reflective 则采用更为精细的多轮执行-观察-反思工作流，允许智能体在最多 20 次迭代中持续分析中间结果、调整分析策略，通过对话式反馈循环实现渐进式求精。这种反思性执行范式与 ReAct 框架的思路一脉相承，但将其落地于高度专业化的气象数据分析场景，在每轮迭代中智能体均需对气象数值结果进行科学解读并决策下一步操作。第三个核心组件是 **ZephyrusBench**，一个涵盖 2,158 个问答对、跨越 46 类不同气象任务的综合评测基准。这些样本按难度分为三层：简单级（699 条）涵盖基础数据查询与极值检索；中等级（526 条）涵盖气象预报与复杂分析流程；困难级（933 条）则包含极端天气事件检测、气候预测及反事实推理等高阶任务。ZephyrusBench 的样本来源兼顾人工标注（1,677 条）与半自动化生成（481 条）——后者通过从气象学文本中抽取可核验声明、转化为问答模板、再经 LLM 代码生成与 ERA5 数据验证的流水线自动构建，具备良好的可扩展性。

在实验评估方面，研究团队以 GPT-5-Mini、GPT-5-Nano、Gemini 2.5 Flash 以及 gpt-oss-120b 四款前沿 LLM 作为底层推理引擎，在 ZephyrusBench 上进行了系统性对比。实验采用多维度评估体系：数值型答案使用标准化中位数绝对误差（阈值 $< 0.05\sigma$）；位置型答案结合地球移动距离（Earth Mover's Distance）与模糊字符串匹配；描述性答案则通过 LLM 辅助的声明抽取与 Precision/Recall F1 框架评分；布尔类问题采用精确匹配。整体结果表明，Zephyrus 智能体相较于纯文本基线有显著提升：以 GPT-5-Mini 为例，Zephyrus-Reflective 达到 54.7% 的正确率，而文本基线仅为 19.9%，提升幅度高达 34.8 个百分点；所有测试模型均获得 28.6 至 35.4 个百分点不等的增益，最高改善幅度达到 44 个百分点。分任务难度来看，简单任务的正确率介于 78.7% 至 88.1% 之间，中等任务为 39.9% 至 50.5%，而困难任务上智能体与纯文本基线相当，说明在涉及抽象推理与长程气候分析的任务上，当前框架仍面临瓶颈。地理位置预测任务上，Zephyrus-Reflective 搭配 GPT-5-Mini 取得了 86.6% 的准确率；数值类任务的标准化绝对误差中位数接近 0；而描述性任务（如全球气候预测报告生成）的得分则较低，最高讨论得分仅为 0.177，美国本土预测约为 0.31，全球 3 个月气候预测几乎所有模型均完全失败。消融实验进一步揭示，多轮反思机制相较于直接生成方案带来了 3.5 至 6.2 个百分点的稳定提升，验证了迭代细化策略在气象分析任务中的有效性。

综合而言，Zephyrus 是气象科学与智能体 AI 的一次有意义的交叉创新。该工作的学术意义在于首次系统性地将 LLM 的语言推理能力与气象专业工具链深度耦合，并提供了可复现的基准评测体系，为后续气象 AI 智能体研究奠定了重要的方法论基础。从实践角度看，ZephyrusWorld 提供的统一 Python API 降低了气象数据分析的技术门槛，使非编程背景的气候科研人员也能通过自然语言驱动复杂的气象分析流程。该研究也清醒地指出了当前局限：框架在简单数据查询与短期预测任务上表现优异，但面对需要深层气象领域知识的复杂推理任务（如极端天气成因分析、季节性气候预测报告撰写）时仍力不从心。未来的研究方向包括引入更大规模的气象领域训练数据以微调专用智能体、探索更强的观测数据到科学推理的转化机制，以及扩展工具链覆盖更多气象子领域（如海洋、冰川等）。代码与基准数据集已在 GitHub（Rose-STL-Lab/Zephyrus）公开发布，为社区的后续研究提供了坚实的起点。
