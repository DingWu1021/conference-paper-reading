# Kimi-Dev: Agentless Training as Skill Prior for SWE-agents

> **Venue**: ICLR 2026 | **ArXiv**: 2509.23045 | **KimiDev**

软件工程自动化是大语言模型（LLM）应用的重要前沿方向，SWE-bench 作为核心评测基准，已成为衡量模型能否在真实代码仓库中解决 GitHub issue 的标准。在现有解决方案中，学界逐渐分化出两种主流范式：一是以多轮交互为核心的 SWE-Agent 框架，允许模型反复与环境交互、执行命令、观察反馈、逐步修复 bug；二是基于工作流的 Agentless 方法，将整个修复任务分解为若干单轮可验证的步骤（如文件定位、代码编辑、测试生成），以结构化流程替代自由式交互。长期以来，这两种范式被视为相互对立的设计哲学：Agent 方法灵活性强但训练难度大，面临稀疏奖励与信用分配的困境；Agentless 方法可控性强、易于优化，但缺乏跨轮交互的能力。Kimi-Dev 的核心洞见在于，这两种范式其实并非互斥：经过推理密集型 Agentless 训练所习得的技能先验（Skill Prior），包括代码定位、代码编辑与自我反思等能力，能够有效迁移并赋能多轮 SWE-Agent 框架，从而实现两种范式之间的能力桥接。

Kimi-Dev 基于 Qwen 2.5-72B-Base 构建，整体训练流程分为四个阶段。首先是领域中期训练（Mid-Training），在约 150B token 的精心筛选数据上进行，其中包括约 50B token 来自 GitHub PR 的自然代码差分（natural diffs）、约 20B token 的 PR 提交序列，以及约 20B token 经过 4 倍上采样的合成推理与 Agent 交互模式数据。训练过程严格执行数据去污染，排除了所有 SWE-bench Verified 中涉及的代码仓库，保证评测的公正性。第二阶段为冷启动有监督微调（Cold-Start SFT），使用约 2000 条来自 DeepSeek R1 的推理轨迹对模型进行监督微调，激活模型的长链式思维（long chain-of-thought）能力，使其能够在复杂的代码问题上展开深入的逐步推理。第三阶段为强化学习（RL），采用基于 REINFORCE 的策略梯度优化方法，专注于代码编辑阶段的优化，使用纯结果奖励（outcome-based reward，取值 0 或 1，由实际执行结果决定）。训练中引入自适应课程学习，从可解的简单样本出发逐步扩展难度，并在后期阶段引入正样本强化策略。为支持大规模并行训练，Kimi-Dev 构建了基于 Kubernetes 的沙箱基础设施，支持同时运行超过 10,000 个并发实例。

在 Agentless 推理框架的设计上，Kimi-Dev 采用了两个互补的角色：BugFixer 负责定位相关文件并生成代码补丁，TestWriter 负责生成能够复现 bug 并验证修复效果的单元测试。两者均依赖于文件定位与代码编辑技能。在测试时，系统采用自我博弈（Test-Time Self-Play）策略：针对每个问题生成 40 个补丁和 40 个测试用例，并通过执行交叉评分选出最优补丁。评分函数为

$$S_i = \frac{\sum_j FP(i,j)}{\sum_j F(j)} + \frac{\sum_j PP(i,j)}{\sum_j P(j)}$$

其中 $FP(i,j)$ 衡量补丁 $i$ 在测试 $j$ 上的失败转通过率，$PP(i,j)$ 衡量保持通过的稳定性，$F(j)$ 和 $P(j)$ 分别为各测试的归一化因子。这一基于执行的评分机制相比朴素的多数投票显著提升了最终选择质量。

在向 SWE-Agent 范式迁移的阶段，研究者仅使用 5,016 条公开可用的 SWE-Agent 轨迹对 Kimi-Dev 进行监督微调适配（SFT Adaptation），无需额外强化学习训练，即可将 Agentless 阶段习得的技能先验高效迁移至多轮 Agent 交互场景。

实验结果充分验证了 Kimi-Dev 的有效性。在 Agentless 工作流模式下，Kimi-Dev 在 SWE-bench Verified 上实现了 60.4% 的通过率，成为开放权重工作流方法中的最优结果，显著超越 Llama3-SWE-RL（70B，41.0%）、SWE-SWISS（32B，58.2%）以及 DeepSeek-R1-0528（671B，57.6%）。值得注意的是，Kimi-Dev 仅有 72B 参数，却能够超越参数量近十倍的 DeepSeek-R1-0528，体现了其训练方法的高效性。在 SWE-Agent 模式下，经过 5k 条轨迹的 SFT 适配后，Kimi-Dev 驱动的 Agent 达到了 48.6% pass@1，与 Claude 3.5 Sonnet（241022 版本，49%）持平，而 pass@10 更进一步达到 74.0%，展示出模型在多次尝试时的强大潜力。

消融实验进一步揭示了技能先验的核心价值。对比分析表明，经过 RL 训练的先验模型在所有规模的微调数据下均显著优于基础模型，尤其是在数据极度稀少的情况下（仅 200 条 Agent 轨迹），RL 先验依然保持明显的性能优势。这意味着 Agentless 阶段的强化学习所建立的技能先验大幅降低了 Agent 适配所需的数据量，实现了数据高效迁移：基础模型需要 $1.5 \times 2^{28}$ SFT token 才能达到 RL 先验仅需 $2^{23}$ token 便可实现的性能水平，数据效率提升约 32 倍。在长时程推理能力方面，采用 RL 先验的 Agent 能够在超过 70 轮的多轮交互中持续改善表现，而基础模型在 50-60 轮时便开始退化，说明 Agentless 阶段培养的链式推理能力能够有效迁移至扩展的多轮交互中。从 BugFixer 阶段分析来看，RL 先验模型在第三阶段的 bug 解决数量（605 例）远高于基础模型（484 例），说明其自我反思与纠错能力显著增强。此外，测试时自我博弈的并行扩展现象（Parallel Scaling）表明，无需额外训练，单纯通过增加并行采样对数量即可持续提升性能，从 1×1 的 48.0% 扩展到 40×40 的 60.4%。

Kimi-Dev 的研究意义在于从根本上重新审视了软件工程任务中 Agentless 与 Agent 范式之间的关系。传统观点将二者视为对立的设计选择，而本文以严格的实验证明，结构化的工作流训练所习得的细粒度技能——代码定位的准确性、代码编辑的精确性、以及基于执行反馈的自我反思——能够作为通用的技能先验，有效泛化到多轮自由交互的 Agent 场景。这不仅为小参数量开放权重模型追赶前沿闭源模型提供了新的可行路径，也为学界提供了一个关于"结构化训练如何为灵活智能体奠基"的具有说服力的存在性证明。Kimi-Dev 的模型权重与训练代码已在 GitHub（MoonshotAI/Kimi-Dev）开源发布，有望推动软件工程自动化领域的进一步研究。在未来工作方向上，作者指出了若干值得探索的问题：更细粒度的 RL 奖励信号设计、测试生成多样性的提升（当前 TestWriter 存在一定的误报率）、以及将该范式推广到软件工程之外的更广泛 Agent 任务场景。
