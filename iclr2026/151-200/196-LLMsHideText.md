# LLMs Can Hide Text in Other Text of the Same Length

> **Venue**: ICLR 2026 | **ArXiv**: 2510.20075 | **LLMsHideText**

这篇论文由 Antonio Norelli 与 Michael Bronstein 合作完成，于 2025 年 10 月发布在 arXiv，并被 ICLR 2026 收录。研究的核心命题令人深感震惊：一段有意义的文本，可以完整地隐藏在另一段长度完全相同、内容截然不同却同样流畅自然的文本之中。作者们以此揭示了大型语言模型所蕴含的一种此前未曾被系统研究过的隐写能力，并将其定性为对人工智能安全与数字信任体系的根本性威胁。

研究的动机根植于当前大模型时代对"文本真实性"这一基础假设的质疑。在传统的书面交流中，人们默认文字是作者意图的直接表达——即文本的表层含义与作者的真实意图是一致的。然而，大型语言模型的出现已经开始动摇这一假设：模型生成的文字未必反映任何真实意图，而只是统计意义上最合理的token序列。这篇论文更进一步，证明了语言模型可以被主动用于构造一种"双重文本"——其表层内容是一段完全正常、无害的文字，而其深层编码的却是一段完全不同的秘密信息。这种能力在技术层面意味着"文本与作者意图的根本性解耦"，在安全层面则意味着现有的内容审查、对齐过滤等防护机制存在被系统性绕过的可能性。

论文提出的核心技术贡献是一个被命名为 **Calgacus** 的隐写协议。协议的基本思路利用了语言模型在生成每个 token 时所维护的概率分布，具体分为三个阶段。首先是排名记录阶段：给定一段需要隐藏的"载荷文本"（payload text）$e$ 以及一个共享的秘密提示词（secret prompt）$k$，发送方使用语言模型对载荷文本的每个 token $e_i$ 进行排名查询——即在模型以 $k$ 以及前文 $e_{<i}$ 为条件时，$e_i$ 在概率分布中排第几位，记录这一排名序列 $r = (r_1, r_2, \ldots, r_n)$。其次是载体文本生成阶段：发送方另行构造一个不同的起始提示词（cover prompt），以此为条件让模型生成一段载体文本（stegotext）$s$——但生成时并非按照标准采样策略选取最高概率的 token，而是在每一步强制选取概率排名恰好为 $r_i$ 的那个 token。由于排名 $r_i$ 来自于原载荷文本在另一个分布下的排名，而语言模型对合理内容的概率分布通常集中在低排名区间，故生成的 $s$ 依然保持语言流畅性。最后是解码阶段：接收方持有相同的语言模型与秘密提示词 $k$，对载体文本 $s$ 逐 token 反查其在各自条件分布中的排名，从而精确还原排名序列 $r$，进而以 $r$ 为索引从条件分布中选出对应 token，完整恢复原始载荷文本 $e$。整个协议的编解码均可在普通笔记本电脑上、使用参数量仅 80 亿的开源模型（如 LLaMA 系列）在数秒内完成，极大地降低了实际部署的门槛。

从信息论的角度理解，该协议实现了近满容量的隐写编码。由于每个 token 对应的排名 $r_i$ 可以取 $[1, V]$（$V$ 为词表大小）中的任意值，理论上每个 token 位置可编码 $\log_2 V$ 比特的信息。然而，载荷文本所使用的具体排名通常集中在低值区间（因为合理文本本身就倾向于高概率 token），这意味着对应的载体 token 也大概率是高概率 token，从而保证了载体文本的流畅性。反之，当载荷文本包含低概率（高排名）token 时，对应的载体 token 也会落入低概率区间，导致"低熵 token 资源浪费"问题，这是协议的主要局限之一，会使载体文本的整体概率略低于自然生成文本。

在实验设计方面，作者们对 Calgacus 协议从多个维度进行了评估。在生成质量方面，实验通过人类评估者与 LLM 评估者双重打分，验证了生成的载体文本在流畅性、连贯性、合理性方面与真实文本相当。以隐藏摘要级别长度的文本为例，人类评估者难以区分哪段是"原文"哪段是"载体文本"。在可检测性方面，研究发现虽然人类几乎无法识别载体文本，但 LLM 自身可以通过比较两段文本的联合概率来在统计意义上区分原文与载体文本——后者的整体概率通常略低，因为低熵 token 问题会系统性地压低载体文本的似然。这意味着存在一种基于 LLM 的检测方法，但其错误率也不可忽视，且在边界样本上存在较高的混淆，提供了一定程度的"合理可否认性"（plausible deniability）。在适用性方面，实验表明协议的效果依赖于载荷文本与模型训练分布的相符程度：对标准英语内容效果优秀，但对罗曼斯科方言或高度专业化领域内容质量有所下降，体现了模型能力边界对隐写效果的制约。

论文还专门讨论了一个尤为值得警惕的应用场景：企业可以在对外部署一个经过对齐与安全过滤的"合规模型"的同时，将一个未经对齐的"不受限模型"的真实回答编码隐藏在合规模型的表面输出之中。在接收端，持有秘密提示词的特定用户可以从看似无害的合规回答中完整解码出真实、未经过滤的内容。这种攻击方式从根本上架空了当前主流的 AI 安全防护思路——无论是基于输出内容审查的过滤系统，还是基于人类反馈强化学习（RLHF）的对齐训练，都无法检测到这种隐蔽的信息传递渠道。

从哲学层面，作者进一步将这一发现延伸至对"幻觉"概念的重新定义。传统上，LLM 的幻觉被理解为模型输出与事实不符的错误。而本文认为，更本质的问题是语言模型的输出本身就不携带"作者意图"——模型既可以言之凿凿地输出事实，也可以完全相同地输出隐写编码后的载体文本，两者在表面上无法区分。由此引出一个深刻的认识论问题：LLM 究竟"知道"什么？其输出的内容是否具有可信赖的认识论地位？这一问题的回答将深刻影响人类社会对 AI 生成内容的信任基础。

综合来看，这篇论文的研究意义体现在多个层面。在技术层面，它首次系统性地证明了现有规模的开源 LLM 足以支撑高质量的文本隐写，并给出了完整的协议设计与实现（代码已开源于 GitHub）。在安全层面，它揭示了对齐机制的一个根本性盲点：安全对齐针对的是模型的"意图"，但当意图本身可以被隐写技术从表层输出中剥离时，对齐的有效性就成了问题。在社会层面，它对书面通信的信任基础提出了严峻挑战——在 LLM 被广泛用于内容生产的时代，任何一段文字都可能同时是两段截然不同内容的叠加态，这使得"读懂一段文字"这件事在技术上变得前所未有地复杂。作者们以苏格兰历史上的演说家 Calgacus 命名这一协议——据记载，Calgacus 曾在面对罗马征服时发表了著名的"制造荒漠，称之为和平"的演说，而这段演说本身正是通过罗马史家之口得以留存，这种"借敌人之口传递真实之声"的历史典故，与协议"借合规文本传递隐秘信息"的技术本质形成了深刻的隐喻对照。
